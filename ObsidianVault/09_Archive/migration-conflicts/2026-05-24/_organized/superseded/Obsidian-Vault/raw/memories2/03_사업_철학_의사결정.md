# 사업 철학 & 의사결정 기준

---

## 1. 핵심 철학 변화

**초기 관점**: AI = 돈을 벌기 위한 도구
**현재 관점**: AI = 협업 파트너 → 시스템이 작동한 결과로 수익이 따라옴

---

## 2. 핵심 철학 7가지

1. AI는 단순 도구가 아니라 협업 파트너다
2. 돈은 시스템이 제대로 작동한 결과로 따라와야 한다
3. 무작정 아이디어를 벌리는 것보다 실행 가능한 구조를 먼저 만든다
4. 법적 / 보안 / 저작권 문제를 피하는 것이 중요하다
5. 무료 또는 저비용 기반으로 시작 → 수익 발생 후 유료 도구로 점진 확장
6. 자동화 시스템은 반드시 현실 업무와 연결되어야 한다
7. 범용 AI보다 인테리어 특화형 AI가 더 중요하다

## 5. 개발 지시 철학

**재하 = 방향 제시 / 판단 / 컨펌 / 사업 의사결정**
**AI = 그 구조 안에서 실행**

- 승인 전 실코드 생성 절대 금지
- 설계·의사코드·스니펫은 승인 없이 제공 가능
- 구현해 = 승인 키워드

**작업 규모별 처리**

| 규모 | 기준 | 처리 방식 |
|------|------|-----------|
| 소 | 단순 변경 | 즉시 처리 |
| 중 | 다수 파일 변경 | 계획 → 승인 |
| 대 | 아키텍처 변경 | 리서치 → 계획 → 승인 |
| 불확실 | — | 중으로 처리 |

보안 / 결제 / 인증 = 규모 무관 항상 계획 → 승인

---

## 6. 운영 헌장 10조

1. 모든 프로젝트는 현실적으로 구현 가능한 수준에서 시작한다
2. 무료 또는 저비용 MVP를 우선한다
3. 법적 문제, 저작권 문제, 보안 문제를 반드시 고려한다
4. 개발은 Claude Code가 주로 수행하고, Codex가 검증한다
5. AIMY는 전체 구조 / 전략 / 프롬프트 / 검증 기준을 담당한다
6. 모든 중요한 결정은 Obsidian에 기록한다
7. 모든 개발 프로젝트는 데이터 저장 구조를 포함한다
8. 모든 웹앱은 모바일과 데스크탑을 고려한다
9. 모든 시스템은 향후 수익화 가능성을 고려한다
10. 인테리어 실무성과 AI 자동화 연결성을 최우선으로 본다
