## 📋 프로젝트 연표

### 🔷 JH-견적시스템 (인테리어 견적 정규화)

| 항목 | 내용 |
|---|---|
| 목적 | 업체별 다른 Excel 구조를 표준화 |
| 스택 | FastAPI + Next.js 14 + Supabase |
| 에이전트 | 5개 (orchestrator, template_manager, normalizer, classifier, bid_formatter) |

**핵심 문제**: 각 클라이언트 브랜드마다 Excel 컬럼 구조가 달라 자동 처리 불가
→ normalizer 에이전트가 다양한 형식을 표준 스키마로 변환

**보안 인시던트**: `.env` 파일을 채팅에 실수로 업로드
→ API 키 로테이션 실시
→ 전체 팀 보안 프로세스 강화

### 🔷 jh-playwright-agents

| 항목 | 내용 |
|---|---|
| 목적 | Playwright 기반 브라우저 자동화 에이전트 |
| 규모 | 27개 파일, 13개 에이전트 |
| 트랙 | 3개 (DEV 검사 / 비즈니스 자동화 / 배포 전 체크) |
| 특징 | 전역 JH 가이드라인과 **별개** (아키텍처 충돌) |

**충돌 이유**: 자율 연속 실행 vs 승인 기반 워크플로우 — 두 원칙이 충돌
→ jh-playwright-agents는 독립 가이드라인으로 운영
