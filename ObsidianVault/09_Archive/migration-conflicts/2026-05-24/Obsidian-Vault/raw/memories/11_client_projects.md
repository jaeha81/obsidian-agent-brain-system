## 🏢 EONID 브랜드 포트폴리오 대시보드

| 항목 | 내용 |
|---|---|
| 클라이언트 | EONID |
| 목적 | 멀티브랜드 포트폴리오 관리 대시보드 |
| 기술 | React (멀티탭) |
| 납품 | Netlify 배포 패키지 |

### 포함 브랜드 (7탭)
1. **Adidas** 관련 브랜드
2. **Converse** 관련 브랜드
3. **Twosome** 관련 브랜드
4. **SPC 서브브랜드** (여러 개)
5. **경복대** 관련 콘텐츠
6-7. 추가 브랜드 탭

### 디자인
- **색상 스킴**: 다크 올리브 / 라임
- **스타일**: 다크 테마, 브랜드별 구분
- **반응형**: Netlify 배포 최적화

## 📋 클라이언트 프로젝트 원칙

### JH 인프라 분리
- 모든 클라이언트 프로젝트는 JH 개인 인프라와 완전 분리
- 독립 배포 환경 (클라이언트 도메인)
- JH 브랜딩 포함 여부: 협의에 따라

### 납품 프로세스
1. 기획/제안서 작성 (DOCX/PDF)
2. 클라이언트 승인
3. 구현
4. 납품 패키지 (Netlify-ready, Docker-ready 등)

### 보안
- 클라이언트 데이터는 클라이언트 인프라에만 저장
- API 키, 민감 정보는 클라이언트에게 직접 관리 지시
- 재하는 구조 설계 + 구현 제공, 운영은 클라이언트 책임
