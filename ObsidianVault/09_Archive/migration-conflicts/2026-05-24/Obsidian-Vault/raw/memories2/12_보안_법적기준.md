## 2. API Key 보안 이력

**실제 사고 사례**: JH-견적시스템 개발 중 `.env` 파일이 채팅에 업로드됨
→ 키 교체 완료 (이후 재발 방지 규칙 강화)

**규칙**
- 시크릿 / API키 / PII / DB정보 감지 시 즉시 중단 고지
- 외부 API 연동 시 인증정보 전달 방식 검토 후 안전한 방식 제안

## 4. 법적·윤리적 운영 기준

재하는 규정 회피보다 투명성과 기록을 남기는 방향을 선호한다.

| 기준 | 내용 |
|------|------|
| 규정 준수 | 플랫폼 약관 준수 |
| 신뢰성 | 투명한 기록 |
| 데이터 수집 | 고객 동의 기반 |
| 개인정보 | 최소 수집 |
| 자동화 오남용 | 방지 |

**민감 기능 법적 검토 필수 항목**
- 경찰 / 고지서 / 차량번호 / 감시 기능 → 공공기관 권한 및 법적 범위 검토 필수
- 음성녹음 → 명확한 동의 + 안내 필수
- 화면수집 → 명확한 동의 + 안내 필수
- 위치추적 → 명확한 동의 + 안내 필수

## 6. 외부 공유 기준

이 백과사전은 개인 Obsidian Vault 전용이다.

**공유 시 분리 기준**

| 대상 | 포함 항목 |
|------|---------|
| 투자자 / 협력사 | 별도 외부용 요약본 |
| 회사 내부 | 민감한 개인 사업 내용 제거 |
| 고객 | 브랜드 소개 중심 재작성 |
| 개발자 | 필요한 기술 요구사항만 추출 |
| AI (Claude / Codex) | 민감 정보 제거 버전 사용 권장 |

**제거 항목**
- 개인 부채 관련 내용
- 회사 내부 역할과 별도 사업 관계
- 비공개 브랜드 전략
- API Key / Webhook URL
- 개인 연락처
- 내부 개발 방식 세부
- 투자 및 선물옵션 전략 세부
- 민감한 자동화 아이디어
