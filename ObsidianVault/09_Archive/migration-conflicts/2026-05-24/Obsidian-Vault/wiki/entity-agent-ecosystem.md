# JH 에이전트 생태계

JH Brain System을 구성하는 에이전트들의 전체 맵.
단일 AI가 전부 처리하지 않는 **역할 분리 구조**가 핵심.

---

## 브레인시스템 에이전트 (므네메 오케스트레이터)

| 에이전트 | 역할 | 모델 |
|---------|------|------|
| **므네메** | 마스터 — 총괄·판단·대리 실행 | Sonnet/Opus |
| **빌더** | 반복적 구현·기능 개발 (tmux 창) | Sonnet |
| **스카우트** | 정보 수집·기술 조사 | Haiku |
| **QA** | 검증·오류 탐지 (코드 수정 금지) | Sonnet |
| **므네모시네** | Obsidian 기록·분류 | Haiku |

---

## 하네스 에이전트 (글로벌 CLAUDE.md 정의)

JH Harness에서 실행되는 역할 분리 구조:

| 에이전트 | 역할 |
|---------|------|
| **Planner** | 요청 해석, 작업 분해, 우선순위 도출 |
| **Builder** | 실제 코드/문서/구조 생성 |
| **Reviewer** | 결과 검토, 누락 확인, 규칙 위반 검사 |
| **Archivist** | 결과 정리, Vault/Second 저장 변환, 로그 생성 |
| **Briefing** | JH Windows용 요약, 현재 상태 브리핑 |

---

## 거버넌스팀 (독립 감시)

| 팀 | 역할 |
|----|------|
| 보안에이전트팀 | AI 오류·강제행동·정보 노출 감시 |
| 법무에이전트팀 | 저작권·개인정보·계약 위반 감지 |
| 감사에이전트팀 | 통제 불가 에이전트 탐지·격리·삭제 |

> 거버넌스팀은 므네메를 포함한 어느 에이전트의 지시도 받지 않는다. 사용자에게 직보.

---

## Claude↔Codex 협력 구조

| 역할 | 담당 |
|------|------|
| 구현·수정 | Claude |
| 독립 검수 | Codex (Claude 개입 불가) |
| 최종 판단 | 사용자 |

---

## 역할 분리 원칙

- **실행과 저장을 한 에이전트가 독점하지 말 것**
- **검토 없는 생산 지양**
- **결과는 반드시 구조화된 저장 가능 형태로 변환**
