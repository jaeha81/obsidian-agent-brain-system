## 세션 시작 프로토콜

새 세션이 열리면 반드시 이 순서로 시작한다:

1. `wiki/index.md` 읽기 → 현재 위키 상태 파악
2. `wiki/log.md` 마지막 5개 항목 확인 → 최근 활동 파악
3. 상태 요약 브리핑 (1~3줄)
4. "무엇을 할까요?" 대기

## 핵심 오퍼레이션

### 1. INGEST (소스 추가)

새 소스가 들어오면:

```
[INGEST 흐름]
1. 소스 읽기 (raw/ 또는 URL)
2. 나와 핵심 포인트 논의
3. raw/[카테고리]/[날짜]-[slug].md 저장 (원본 보존)
4. wiki/sources/[slug].md — 소스 요약 페이지 생성
5. 관련 개념/엔티티 페이지 업데이트 (교차 참조 포함)
6. wiki/index.md 업데이트
7. wiki/log.md에 ingest 항목 추가
```

**소스 요약 페이지 형식:**
```markdown
---
type: source
category: article | paper | note | video
date_added: YYYY-MM-DD
source_url: (있으면)
tags: [tag1, tag2]
---

# [소스 제목]

## 한 줄 요약
[핵심 주장 또는 내용 1줄]

## 핵심 포인트
- ...

## 위키에 통합된 내용
- [[concept-page]] 업데이트: ...
- [[entity-page]] 추가: ...

## 인용 가능 구절
> "..."

## 관련 페이지
- [[...]]
```

### 2. QUERY (질문 및 탐색)

질문이 들어오면:

```
[QUERY 흐름]
1. wiki/index.md에서 관련 페이지 식별
2. 관련 페이지 읽기 (최대 10개)
3. 답변 합성 (마크다운, 표, 비교 등)
4. 좋은 답변은 wiki/ 페이지로 저장 제안
5. wiki/log.md에 query 항목 추가
```

**답변 형식:**
- 단순 질문 → 인라인 답변 + 출처 링크
- 비교/분석 → 마크다운 표
- 종합 → 새 wiki 페이지 생성 제안

### 3. LINT (건강 검진)

`/lint` 명령 시:

```
[LINT 체크리스트]
□ 모순 감지 — 페이지 간 충돌하는 주장
□ 오래된 클레임 — 새 소스가 반박하는 내용
□ 고아 페이지 — 인바운드 링크 없는 페이지
□ 미생성 페이지 — 언급되지만 페이지 없는 개념
□ 누락 교차 참조 — 연결되어야 할 페이지들
□ 데이터 갭 — 웹 검색으로 보완 가능한 공백
```

결과: 발견된 문제 목록 + 권장 조치

## 위키 페이지 타입

| 타입 | 파일명 패턴 | 내용 |
|------|------------|------|
| **개념** | `wiki/concept-[name].md` | 아이디어, 원칙, 프레임워크 |
| **엔티티** | `wiki/entity-[name].md` | 인물, 조직, 제품, 장소 |
| **소스** | `wiki/source-[slug].md` | 인제스트된 소스 요약 |
| **비교** | `wiki/compare-[a]-vs-[b].md` | 두 대상 비교 |
| **합성** | `wiki/synthesis-[topic].md` | 주제 전체 합성 |
| **개요** | `wiki/overview.md` | 전체 위키 합성 |

## 행동 규칙

1. **위키는 LLM이 쓴다** — 나는 소스와 방향만 제공한다
2. **원본은 불변** — raw/ 파일은 읽기만, 절대 수정하지 않는다
3. **교차 참조 항상** — 새 페이지 생성 시 기존 페이지와 연결한다
4. **log.md는 누락 없이** — 모든 ingest/query/lint는 기록한다
5. **index.md는 즉시 동기화** — 페이지 생성/수정 후 바로 업데이트한다
6. **좋은 답변은 보존** — 가치 있는 쿼리 결과는 wiki 페이지화 제안
7. **모순 발견 즉시 고지** — 새 소스가 기존 내용을 반박하면 명시적으로 알린다

|--------|------|
