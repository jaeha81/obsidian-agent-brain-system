## 3. 개발 워크플로우

```
research.md 작성
    ↓
plan.md 작성
    ↓
재하 승인 ("구현해")
    ↓
구현 시작
```

- **승인 전 실코드 생성 절대 금지**
- 설계 / 의사코드 / 스니펫은 승인 없이 제공 가능
- 보안·결제·인증은 규모 무관 항상 계획 → 승인

## 6. 개발 착수 전 체크리스트

```markdown
## 1. 목적
- 이 앱은 무엇을 해결하는가?

## 2. 사용자
- 누가 사용하는가?
- 어떤 기기에서 쓰는가?

## 3. 핵심 기능
- MVP에 반드시 필요한 기능은?
- 나중으로 미뤄도 되는 기능은?

## 4. 데이터
- 어떤 데이터를 입력받는가?
- 어디에 저장하는가?
- 저장하면 안 되는 데이터는?

## 5. 보안
- 로그인 필요 여부
- 관리자 접근 제한
- API Key 보호
- 업로드 파일 보호

## 6. 자동화
- 어떤 외부 서비스와 연결되는가?
- Webhook이 필요한가?
- Make / n8n / GAS 중 무엇이 적합한가?

## 7. 배포
- Netlify인가 Vercel인가?
- 로컬 테스트 후 배포인가?

## 8. 검증
- Claude Code 구현
- Codex 검토
- AIMY 구조 검증
```
