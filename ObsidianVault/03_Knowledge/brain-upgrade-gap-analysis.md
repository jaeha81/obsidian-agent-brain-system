## 기능별 갭 분석표

### 1. 영구 기억 (Remembers Everything)

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🟡 "remembers everything" (120.6K 영상 캡션) |
| 기술 실체 | 🟢 구현 가능. 대화/작업 로그를 마크다운 파일로 누적 저장 + 인덱스 파일(MEMORY.md류)로 회수. Claude Code 기반에서 표준 패턴. |
| 구현 난이도 | 낮음 |
| 현재 시스템 상태 | ✅ **구현 완료**. `~/.claude/projects/.../memory/` 폴더에 MEMORY.md 인덱스 + 1팩트=1파일 구조로 운영 중. 세션 시작 시 자동 로드됨. |
| 적용 권장 | 메모리 파일 1팩트=1파일, frontmatter(type/description), 인덱스 1줄 포인터 구조 |
| 영상 확인 필요? | 아니오 — 표준 기술로 충분 |

### 2. 실시간 패턴 감지 (Notices Patterns in Real Time)

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🟡 "notices patterns in real time" |
| 기술 실체 | 🟡 "실시간"이 관건. 진짜 상시 백그라운드 감시인지, 아니면 호출 시점에 과거 노트를 스캔하는 것인지 캡션으론 불명. 후자는 쉬움, 전자는 상시 프로세스 필요. |
| 구현 난이도 | 호출형=낮음 / 상시형=중간(스케줄러·watcher 필요) |
| 현재 시스템 상태 | ✅ **호출형 구현 완료** (2026-06-09). `scripts/pattern_scanner.py` — goals/ + memory/ 스캔, 반복 작업·반복 실패·유사 목표 감지. 상시형(watcher 통합)은 별도 승인 필요. |
| 적용 권장 | 1단계로 호출형(작업 기록 시 과거 N개 스캔→반복·실패 패턴 보고)부터. 상시형은 검증 후. |
| 영상 확인 필요? | **예** — "실시간"의 실제 의미 확인 필요 |

### 3. 화면 감시 (Watches Your Screen)

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🔴 "watches your screen" (가장 데모 의존적) |
| 기술 실체 | 🟡 스크린샷 주기 캡처→비전 모델 해석은 기술적으로 가능하나, vault 기반 노트 시스템과는 별개 레이어. 비용·프라이버시·상시 권한 이슈 큼. |
| 구현 난이도 | 높음 (상시 캡처 인프라 + 비전 처리 + 프라이버시 설계) |
| 현재 시스템 상태 | ❌ **미구현**. computer-use MCP로 요청 시 스크린샷은 가능하나 상시 감시 없음. |
| 적용 권장 | **보류.** 단순성 우선 원칙상 핵심 가치(기억+패턴)와 분리. 정말 필요한지부터 검토. |
| 영상 확인 필요? | **예** — 실제 작동하는지 vs 연출인지 확인 필수 |

### 4. 음성 작업 위임 ("Jarvis, wake up")

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🟡 "아침에 일어나 Jarvis에게 작업 지시" (41.2K, 60.1K) |
| 기술 실체 | 🟢 음성→텍스트(STT)→에이전트 호출은 표준. 핵심은 음성이 아니라 "목표 1문장→자율 실행" 루프(이미 골모드 프롬프트로 설계함). |
| 구현 난이도 | 낮음 (음성 레이어는 선택, 핵심 루프는 텍스트로 충분) |
| 현재 시스템 상태 | ⚠️ **텍스트 버전만 구현**. `goalmode-prompt-v1.md`로 목표 위임 루프 설계 완료. 음성 레이어(STT) 없음. 핵심 가치는 텍스트로 이미 제공 가능. |
| 적용 권장 | 음성은 후순위. [[goalmode-prompt-v1]]의 목표 위임 루프가 동일 가치를 텍스트로 제공. |
| 영상 확인 필요? | 아니오 |

### 5. 정적 노트 → 능동 브레인 (Anti-Obsidian)

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🟡 "Obsidian 버려라" (95.2K, 22.5K) — 정적 위키의 한계 비판 |
| 기술 실체 | 🟢 타당한 방향. 노트가 "읽히기만" 하는 게 아니라 다음 실행에 자동 주입·재사용되는 구조. 현재 폴더명(obsidian-agent-brain-system)이 이미 이 방향. |
| 구현 난이도 | 중간 (검색·주입 파이프라인) |
| 현재 시스템 상태 | ✅ **구현 완료**. `context_pack_selector.py`로 요청 텍스트 → 관련 vault 노트 자동 선택·주입. Discord 봇 + claude_cli 에이전트 레이어 존재. Obsidian은 백엔드로 유지됨. |
| 적용 권장 | 노트→에이전트 컨텍스트 자동 주입. 단, Obsidian을 버릴 필요 없이 vault를 백엔드로 유지하며 에이전트 레이어만 추가하는 게 단순. |
| 영상 확인 필요? | 아니오 — 사상은 명확 |

### 6. 비즈니스 엔드투엔드 자동화 (콜드콜·마케팅·이커머스)

| 항목 | 내용 |
|---|---|
| 주장 출처 | 🔴 "비즈니스 전체를 처음부터 끝까지 자동화" (4.2K) |
| 기술 실체 | 🔴 가장 과장된 주장. 부분 자동화는 가능하나 "엔드투엔드"는 마케팅 수사. |
| 구현 난이도 | 매우 높음 / 범위 과다 |
| 현재 시스템 상태 | ❌ **범위 밖 — 의도적 미구현**. |
| 적용 권장 | **제외.** 개인 브레인 시스템의 범위를 벗어남. 스코프 크리프. |
| 영상 확인 필요? | 아니오 — 적용 대상 아님 |
