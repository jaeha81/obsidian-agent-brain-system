# 요즘 유행하는 Loop Engineering!

![thumbnail](https://i.ytimg.com/vi/z-3BRkxQ5GM/maxresdefault.jpg)

## 요약

Boris Cherny(보리스 체니)와 Peter Steinberger(피터 스타인버거)의 언급으로 유행하기 시작한 **Loop Engineering(루프 엔지니어링)**을 "그렇게 어려운 개념이 아니다"라며 시간순 계보로 풀어낸다. 핵심은 세 개념의 구분: **랄프 루프(Ralph Loop)** → **하네스 엔지니어링(Harness Engineering)** → **루프 엔지니어링**. 랄프 루프는 롱러닝 에이전트가 없던 시절, 휴먼-인-더-루프를 없애고 작업이 끝날 때까지 자동 반복시키려 만든 단순무식한 기법으로 **0→10 진행(progress)에 포커스**했고, 컨텍스트 열화(니들 인 헤이스택) 문제 때문에 작업을 잘게 쪼개 결과만 디스크에 저장하고 새 컨텍스트로 다음 작업을 돌리는 게 핵심이었다. 하네스 엔지니어링은 **목표를 향해 조건을 벗어나지 않게 제한하는 스코프 정의**다. 루프 엔지니어링은 **첫 프롬프트는 유지하되, 2·3·n번째 개선용 프롬프트(사람이 반복 입력하던 부분)를 AI로 대체**해 완성도를 끌어올리는 방식이다. 발표자는 "프롬프팅을 아예 하지 말라"는 과장된 해석을 경계하며, 대립이 아니라 **중첩(overlap)**이라고 강조한다.

## 핵심 개념

- **세 개념은 대립이 아니라 중첩**: 랄프 루프 → 하네스 엔지니어링 → 루프 엔지니어링 순으로 등장. 결(층위)이 다르지만 서로 겹친다.
- **랄프 루프(Ralph Loop)**: 가장 먼저 나온 기법. **롱러닝 에이전트/골 개념이 없던 시대**에, PRD·체크포인트마다 끼던 휴먼-인-더-루프를 없애고 "완수까지 알아서 반복"시키려 만든 단순무식한 루프. 포커스는 **완성도가 아니라 0→10 진행(progress)**.
- 랄프 루프의 핵심 테크닉: 컨텍스트가 꽉 차면 성능이 떨어지는 **니들 인 헤이스택(컨텍스트 열화)** 문제 → 작업을 **잘게 쪼개고**, 각 작업 결과만 디스크에 저장해 컨텍스트를 최소화한 뒤 **새 에이전트/새 컨텍스트로 다음 작업** 실행 → 더 빠르고 효율적.
- **하네스 엔지니어링**: 메타프롬프팅/골 안에 이미 정의돼 있는 것 — **어떤 스코프에서 어떻게 진행할지**를 파일로 저장·관리하면 그게 하네스. **0→100 과정에서 무엇을 해도 되고 안 되는지 제한**하는 정의. 루프라기보다 "제약".
- **하네스를 지나치게 디테일하게 짜지 마라**: 상세할수록 시간만 더 걸리고 오히려 결과가 나빠짐. 울트라 코드를 쓰면 **다이나믹하게 하네스가 생겨나므로**, 우리는 그 환경(컨텍스트)만 제공하면 됨 → 요즘은 초상세 하네스를 안 하는 추세.
- **루프 엔지니어링**: "나를 대체(내치)하는 시스템". 매번 직접 프롬프트를 넣지 않아도 됨. 단, **첫 프롬프트는 필요**하다 — 없애야 하는 건 결과를 보고 개선하려고 다시 넣던 **2·3·n번째 프롬프트**.
- 정확한 루프: **Act → Observe → Decide → Repeat**. 루프가 한 번 돌 때마다 개선하려는 기능의 **완성도가 계속 올라가는 과정** (무에서 유를 한 번에 만드는 AGI가 아님).
- 좋은 예시: "경쟁사 B의 A/B 테스트 방식을 그대로 복제해 줘 → x·y·z 요소가 일치할 때까지 반복, 유사도(similarity)를 측정하고 매 반복 결과를 리포트로 정의해 줘" 처럼 **조건·측정 기준을 주고 완성도를 반복 상승**시키는 것.
- **주의**: Boris가 "my job is loops"라 했다고 자기 프로젝트에 무작정 루프를 짜면 옛날보다 효율이 떨어지는 경우가 많다 — **프로젝트 관점(스코프)이 다르기 때문**.

## 언급된 도구/프레임워크

- `Loop Engineering(루프 엔지니어링)` — 개선용 n번째 프롬프트를 AI로 대체해 완성도를 반복 상승시키는 방식. Act→Observe→Decide→Repeat.
- `Ralph Loop(랄프 루프)` — 롱러닝 이전 시대의 자동 반복 기법. 0→10 진행 포커스, 작업 분할+컨텍스트 최소화.
- `Harness Engineering(하네스 엔지니어링)` — 스코프/제약을 파일로 정의·관리. 0→100 과정의 허용·금지 규정.
- `골(goal) / 울트라 코드 / 딥리서치` — 롱러닝·다이나믹 하네스를 생성하는 실행 수단.
- 인물: `Boris Cherny(보리스 체니)`, `Peter Steinberger(피터 스타인버거)` — 루프 엔지니어링 유행의 발화자.
