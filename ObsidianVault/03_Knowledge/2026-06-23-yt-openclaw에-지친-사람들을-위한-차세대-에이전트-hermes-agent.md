# OpenClaw에 지친 사람들을 위한 차세대 에이전트 Hermes Agent

![thumbnail](https://i.ytimg.com/vi/9m4uFY9hV4k/maxresdefault.jpg)

## 요약

오픈 클로우(OpenClaw)에서 에러가 많아 관리가 힘들었던 사람들을 위한 대안으로 **차세대 에이전트 "허미스(Hermes) 에이전트"**를 실제 설치부터 연동·자동화 워크플로까지 시연한다. 발표자는 허미스가 **더 빠르고, 더 세련됐으며, 무엇보다 기억력이 강화되어** 자신의 워크플로를 잘 기억하고 문제가 덜 생긴다고 평가한다. 실행 환경은 **호스팅어(Hostinger) VPS**를 쓰며, 맥북을 사는 것보다 훨씬 싸다는 점(월 8.99 수준, KVM2 모델 추천, 쿠폰 "코드팩토리")을 강조한다. 핵심 시연은 세 가지다: (1) 도커 매니저에서 허미스 에이전트를 띄우고 **오픈 버튼(웹 UI)과 터미널 버튼(CLI)**을 구분해 접근, (2) 내장 넥소스 크레딧 대신 **자신의 OpenAI 구독을 OAuth로 직접 연동**(터미널에서 model 명령 → OpenAI Codex 선택 → 로그인 코드 붙여넣기 → GPT 5.5, 패스트 모드), (3) **텔레그램 봇 연동**(BotFather `/newbot` → HTTP API 토큰, userinfobot으로 ID 확보)을 에이전트에게 직접 물어보며 진행. 마지막으로 **위스퍼(Whisper) 로컬 모델로 SRT 자막을 생성**해 쇼츠에 노래방식(빨간 글자가 말에 따라 이동하는) 자막을 자동으로 입히는 실전 파이프라인을 보여준다. 발표자는 **"어떻게 할지 모르면 이제부터는 그냥 에이전트한테 바로 물어보는 습관을 들여라"**는 태도를 반복 강조한다.

## 핵심 개념

- **트렌드는 이미 에이전트로 넘어감**: 오픈 클로우(OpenClaw)를 한 번도 안 써본 사람도 많지만, 발표자는 차세대 대안인 **허미스 에이전트**를 권한다. 이름은 에르메스/민스/허미스 등으로 다양하게 불리지만 발표자는 "민스(허미스) 에이전트"로 지칭.
- **허미스의 강점 = 기억력**: 더 빠르고 세련됐으며, **워크플로를 잘 기억하고 문제가 덜 생긴다**. 오픈 클로우는 에러가 많아 관리가 힘들었던 반면 허미스로 옮긴 뒤 만족했다는 개인 경험.
- **실행 환경은 컴퓨터/클라우드 하나면 됨**: 맥북을 써도 되고 클라우드 한 대를 돌려도 됨. 발표자는 **호스팅어 VPS** 사용. 맥북 최소 스펙(약 900달러)에 비해 월 8.99 수준이라 **100개월을 써도 비슷한 가격** → 인프라 관리·설치 부담 없이 테스트하기 좋음.
- **KVM2 모델 추천**: KVM1은 CPU 원코어·4G라 다소 부족할 수 있음. 차이가 얼마 안 나니 **KVM2 권장**. 결제 시 쿠폰 "코드팩토리"로 추가 할인.
- **AI 연동 두 갈래**: (a) 연동이 귀찮으면 **ready-to-use AI(넥소스 크레딧 구매)**로 별도 연동 없이 사용, (b) 이 영상의 방법은 **본인의 OpenAI 구독을 오어스(OAuth)로 직접 연결** — 커맨드 몇 개면 됨.
- **웹 UI vs 터미널 구분이 중요**: 도커 매니저 → 프로젝트에서 허미스 에이전트가 뜨면, 오른쪽 **오픈 버튼 = 웹 UI 진입**(아이디/비밀번호로 사인인), **터미널 버튼 = 도커 이미지 내부 CLI 진입**.
- **UI 구성**: **채팅**(그냥 대화), **세션**(대화 기록 저장소), **칸반**(작업 진행을 라이브로 지켜보는 구경용). 처음부터 전부 파악하려 하지 말 것.
- **OpenAI 구독 연동 절차**: 터미널에서 `hermes` + `model` 명령 → 기본값 넥소스 AI 게이트웨이를 **OpenAI로 선택** → "Codex CLI or OpenAI API" 항목에서 **구독제 사용 시 OpenAI Codex를 반드시 선택** → 로그인 창 링크 복사·붙여넣기 → 구독 계정 선택 → **링크 밑의 코드를 붙여넣기** → GPT 5.5 선택. **패스트 모드** 토글 시 더 빠른 응답.
- **모르면 에이전트에게 직접 물어봐라**: 텔레그램 연동 방법도 에이전트에게 "너랑 텔레그램 연동하려는데 어떻게 하면 돼"라고 물으면 단계를 알려줌. **"이제부터는 그냥 애인(에이전트)한테 바로 물어보는 습관을 들여야 된다."**
- **텔레그램 봇 연동 절차**: BotFather 검색 → `/newbot` → 봇 이름 지정(겹치지 않게 랜덤하게) → **HTTP API 토큰 복사·저장** → userinfobot에서 `/start`로 **내 아이디 확보·저장** → 값(토큰·ID)을 넣으면 게이트웨이가 시작되고 텔레그램에서 바로 대화 가능("안녕" → "안녕").
- **환경변수 넣기 선택**: 토큰을 AI에 넘기기 싫으면 터미널의 환경변수 공간에 직접 입력, 아니면 그냥 대화창에 넣어도 됨(발표자는 그냥 넣는 편).
- **쇼츠 자막 자동화 파이프라인**: 자막 넣기 같은 사소한 작업 자동화가 만족도가 큼. **큰 파일은 텔레그램이 인식 못 하니 구글 드라이브로 전달**(꿀팁) → "**위스퍼 로컬 모델로 SRT 생성**" 요청 → SRT(시간 대비 글자) 기반으로 **큰 흰색 중앙 자막 + 검정 테두리 + 말에 따라 빨간 글자가 이동(노래방식)** 자막을 자동 생성. 에이전트가 스크립트(실제 발화 텍스트)와 편집된 영상을 반환.

## 언급된 도구/프레임워크

- `Hermes Agent(허미스/에르메스/민스 에이전트)` — 오픈 클로우의 차세대 대안 에이전트. 더 빠르고 세련됨, 강화된 기억력으로 워크플로 기억·에러 감소.
- `OpenClaw(오픈 클로우/오픈 클로드)` — 발표자가 이전에 쓰던 도구. 에러가 많아 관리가 힘들었다고 언급.
- `Hostinger(호스팅어) VPS` — 허미스 실행용 클라우드 서버. 영상 설명 링크 + 쿠폰 "코드팩토리"로 할인.
- `KVM2 / KVM1` — 호스팅어 VPS 모델. KVM1은 원코어·4G라 부족, **KVM2 추천**.
- `넥소스(Nexos) 크레딧 / ready-to-use AI` — 별도 AI 연동 없이 바로 쓰는 내장 크레딧 옵션.
- `OpenAI Codex CLI / OpenAI API` — 본인 구독 연동 경로. 구독제는 **Codex 선택 필수**, OAuth 로그인.
- `GPT 5.5` — 연동 후 선택한 모델. 패스트 모드 토글 가능.
- `Telegram BotFather / userinfobot` — 텔레그램 봇 생성(`/newbot`, HTTP API 토큰)과 사용자 ID 확보(`/start`).
- `Google Drive` — 큰 영상 파일 전달용(텔레그램 용량 한계 우회).
- `Whisper(위스퍼) 로컬 모델 / SRT` — 로컬 음성인식으로 SRT 자막 생성 → 노래방식 자막 자동화.
