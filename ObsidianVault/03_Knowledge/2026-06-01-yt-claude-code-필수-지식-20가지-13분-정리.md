# Claude Code 필수 지식 20가지 13분 정리!

![thumbnail](https://i.ytimg.com/vi/sfCm60LVVuI/maxresdefault.jpg)

## 요약

클로드 코드를 200% 활용하기 위한 핵심 슬래시 명령·기능 20여 가지를 13분에 압축 정리한다. **딥 리서치**(소스 수집→크로스체크→검증→생성의 페이즈를 스텝바이스텝으로 실행해 잘못된 정보를 걸러냄), **울트라 코드**(가장 높은 에오트 레벨, 다수 에이전트를 동시 생성해 분석·수정·검증·테스트를 병렬 수행), **배치**(5~30개 워커를 병렬 생성하는 반복작업 최적화), **다이나믹 워크플로**(복잡한 목표를 조사·실행·검증·병합까지 논리적으로 처리) 등의 차이를 명확히 구분해 준다. 이어 **백그라운드(BG)**, **울트라 플랜**(웹에서 실행되는 고완성도 플래닝), **울트라 리뷰**(엑스트라 유세지 필요, 회당 약 $5), **골(goal)**(성공 조건 명시가 핵심 — 메타프롬프팅 권장), **스킬 크리에이터**(스킬 자동 생성+벤치마킹+deprecate 시점 안내), **퓨어 퍼미션스 프롬프트**(과거 승인 이력 기반 allowlist 자동 생성), **BTW**(세션을 더럽히지 않는 질문/포크) 등을 시연한다.

## 핵심 개념

- **딥 리서치**: 요청 시 워크플로가 실행되고 `/workflows`로 진행 스텝 확인 가능. 소스 수집 → 크로스체크 → 검증 → 결과 생성의 각 페이즈를 스텝바이스텝 설계 → **검증 페이즈에서 잘못된 정보를 걸러냄**. 진지한 조사에 필수.
- **울트라 코드**: 에오트(effort) 레벨 중 최상. xHigh + 워크플로가 합쳐져 복잡 작업을 단계별로 설계. 다수 에이전트 동시 생성 → 정보 수집·고강도 리즈닝·동시 코드 수정·검증·테스트까지 완결. 실행 시 레인보우 UI.
- **배치(`/batch`)**: "5~30개 워커를 병렬로 무조건 생성". **같은 패턴의 반복 작업**을 여러 에이전트로 나눠 병렬 처리.
- **다이나믹 워크플로**: 복잡한 목표를 정하면 조사·실행·검증·결과 합치기까지 병렬로 처리. **복잡성/논리 비중이 높을수록 유리**. (배치=반복작업, 다이나믹=복잡·논리 작업)
- **백그라운드(`BG`)**: 실행을 백그라운딩하고 터미널을 프리업. `클로드 에이전트`로 패널 진입해 세션 리뷰·복귀. 마우스 클릭 반응.
- **울트라 플랜**: 슬래시 없이 "울트라플랜"만 쳐도 됨. 세션이 아닌 **웹으로 작업을 쏴** 터미널 해방 + 플래닝 최적화로 더 정확·완성도 높은 플랜. 토큰 더 소모.
- **울트라 리뷰**: **엑스트라 유세지 활성화 필요**(추가 비용). 앤트로픽이 "돈 낼 가치 있다"고 주장. **회당 약 $5** — 프로젝트 가치에 따라 판단.
- **골(`/goal`)**: 프롬프트를 대충 쓰면 안 됨. **변경 대상·성공 조건을 정확히 명시**해야 장시간 고완성도 작업 가능. "이 프로젝트를 X로 변경할 골 프롬프트를 작성해 줘"로 **메타프롬프팅** 추천 → 목표·배경·건드릴 것/안 될 것·제약·성공 기준까지 상세 생성.
- **스킬 크리에이터**: "UI 검증 스킬 만들어 줘"로 스킬 자동 생성. **벤치마킹**·리소스 절감량 제시, **현재 모델에 이미 내장된 능력인지(=deprecate 시점) 안내**. 같은 세션에서 진행한 프로세스를 그대로 스킬화하면 정확도↑.
- **퓨어 퍼미션스 프롬프트**: 스킵 퍼미션스 모드를 싫어하는 사람용. 과거 대화·승인 이력을 분석해 **allowlist를 자동 생성**, 반복 권한 질문 제거 → 더 빠른 바이브 코딩.
- **BTW**: 세션을 더럽히지 않고 질문. 새 채팅창 생성, `F`로 **포크**(기존 대화 유지한 채 분기).

## 언급된 도구/프레임워크

- `딥 리서치 / /workflows` — 수집→크로스체크→검증→생성 페이즈 기반 조사, 진행 스텝 확인.
- `울트라 코드` — 최상위 에오트 레벨, 다수 에이전트 병렬 분석·수정·검증·테스트.
- `/batch` — 5~30 워커 병렬 생성, 반복작업 최적화.
- `다이나믹 워크플로` — 복잡·논리 목표를 조사·실행·검증·병합까지 병렬 처리.
- `BG (백그라운드)` — 실행 백그라운딩 + 세션 패널 관리.
- `울트라 플랜` — 웹 실행 고완성도 플래닝(토큰 다소비).
- `울트라 리뷰` — 유료(≈$5/회) 고가치 코드 리뷰.
- `/goal` — 성공 조건 명시형 장시간 작업. 메타프롬프팅 권장.
- `스킬 크리에이터` — 스킬 자동 생성 + 벤치마킹 + deprecate 안내.
- `퓨어 퍼미션스 프롬프트` — 승인 이력 기반 allowlist 자동 생성.
- `BTW` — 세션 오염 없는 질문 + 포크.

## 우리 시스템 적용 포인트

> JH 브레인 시스템 맥락에서 이 영상의 아이디어를 어떻게 쓸지.
> (Bucky=AI 오케스트레이터, Obsidian Vault=지식두뇌, AgentBus=에이전트 통신, knowledge-distiller=증류 파이프라인, 목표=1인 창업 레버리지)
- [ ] **배치 vs 다이나믹 워크플로 구분**을 Bucky 라우팅 규칙에 반영 — 반복성 높은 작업(예: 백로그 노트 대량 정제)은 배치형 병렬, 복잡·논리 작업은 다이나믹형으로 분기.
- [ ] **골(goal)의 성공 조건 명시 + 메타프롬프팅** → Bucky 패킷의 `done_when`/`verification`을 골 프롬프트로 자동 생성하는 헬퍼 검토.
- [ ] **퓨어 퍼미션스 프롬프트** → 우리 `.claude/settings` allowlist(fewer-permission-prompts 스킬)와 동일 아이디어, 승인 이력 기반 자동 확장 가능성.
- [ ] **딥 리서치의 검증 페이즈** → knowledge-distiller에 "크로스체크·검증 후 정제"를 명시 단계로 삽입, 잘못된 정보 유입 차단.
- [ ] **스킬 크리에이터의 deprecate 안내**(모델 내장 능력화 판단)를 우리 jh-skill 관리에 도입 — 불필요해진 스킬 정리 기준.
