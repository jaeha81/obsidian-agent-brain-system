# AI에게 질문하기 전에 이것부터 하세요 | 옵시디언 + 헤르메스 에이전트 + Graphify로 나만의 지식 쌓기 [LLM 위키]

![thumbnail](https://i.ytimg.com/vi/hglx5cNckAo/maxresdefault.jpg)

## 요약

1. AI에게 질문할 때마다 자료를 처음부터 다시 찾아야 하는 비효율을 해결하기 위해, 오픈소스 도구 **Graphify**를 옵시디언 + 헤르메스 에이전트 기반 LLM 위키에 추가하는 방법을 소개한다.
2. Graphify는 흩어진 자료 수십~수백 개를 통째로 읽어 개념 간 연결 관계를 **지식 그래프** 한 장으로 자동 생성하며, 에이전트가 하나씩 정리하는 것보다 빠르고 저렴하다.
3. 내가 직접 관점을 담은 핵심 노트는 유지하면서, 그 옆에 Graphify로 만든 **참고 자료 층**을 별도 폴더로 쌓아 지식의 폭을 빠르게 확장하는 구조를 실습한다.



## 우리 시스템 적용 포인트

- [x] **Graphify 레이어 도입 검토**: 흩어진 자료 수십~수백개를 통째로 읽어 지식그래프 자동생성 — 현재 gbrain/InfraNodus와 역할 비교 후 도입 여부 결정 필요
- [x] **지식 2층 구조 원칙 채택**: 내가 직접 관점을 담은 핵심노트(Wiki/) + Graphify 참고자료 층(별도 폴더) 분리 운영 → JH 시스템 `04_Wiki/` + `01_RAW/` 분리 원칙과 일치, 명문화 권장
- [x] **LLM 위키 사전 질의 습관화**: AI에게 질문하기 전 먼저 Obsidian 볼트 내 관련 지식을 그래프로 조회 → 버키 채널에서 gbrain.query() 사전 조회 규칙(2026-06-28 추가)과 직접 연결
- [ ] **보류**: Graphify 도입 여부는 gbrain + InfraNodus 현행 구조와 충돌 가능성 있으므로 별도 검토 후 결정
