# 클로드 디자인 대규모 업데이트 총정리 — Design-sync · 업그레이드된 에디터 · Vercel 배포 · 늘어난사용 토큰

![thumbnail](https://i.ytimg.com/vi/VVvyGafmR_k/maxresdefault.jpg)

## 요약

클로드 디자인(Claude Design)의 대규모 업데이트를 직접 써보고 정리한 영상이다. 기존 클로드 디자인은 (1) 토큰을 너무 빨리 소진하고, (2) 디자인 에디터 기능이 약해 박스 크기 변경조차 프롬프트로 해야 했으며, (3) 클로드 코드와 클로드 디자인 사이에서 디자인 시스템이 동기화되지 않는 세 가지 약점이 있었다. 이번 업데이트는 클로드 코드의 코드베이스 디자인 시스템을 통째로 클로드 디자인으로 끌어오는 'Design-sync'(슬래시 `/design-sync`)로 이 오래 묵은 문제를 해결했고, 피그마처럼 손으로 크기·위치를 조정하는 새 에디터, Vercel 통합 원클릭 퍼블릭 배포, 그리고 클로드 코드/코워크와 통합된 토큰 사용량 공유까지 담았다.

## 핵심 개념

- 클로드 디자인은 출시 당시 피그마 주가를 7% 떨어뜨릴 만큼 충격을 줬다 — 수작업 디자인을 프롬프트/AI만으로, 그것도 "AI스럽지 않은" 디자이너급 퀄리티로 대체했기 때문.
- 기존 3대 약점: 과도한 토큰 소진, 약한 에디터(HTML 기반이라 세부 조정이 어려움), 코드↔디자인 간 디자인 시스템 비동기화.
- **Design-sync (핵심 신기능)**: 클로드 코드가 디자인 시스템의 '원천(source of truth)'이 되고, 코드베이스의 디자인 토큰·컴포넌트를 클로드 디자인으로 동기화한다. 업계가 수십 년 풀지 못한 "디자인 측 원천 ≠ 개발 측 원천" 문제에 단일 원천 동기화라는 해법을 제시.
- **새 에디터**: 프롬프트 없이 그 자리에서 크기·위치를 직접 조정 → 속도·비용(토큰) 절감.
- **토큰 통합**: 예전엔 클로드 디자인만 별도 사용량 제한이 있었으나, 이제 클로드 코드·코워크와 통합되어 사용량을 공유 → 실질 사용 가능량 증가.
- **Vercel 내보내기**: 예전엔 팀 내부 공유만 됐지만, 이제 Vercel 통합으로 퍼블릭 웹에 바로 배포 → 사용자 테스트 대상에게 링크 전달 가능.
- 실행 흐름: 클로드 코드에 디자인 시스템 레포가 있어야 하며(예: GitHub의 오픈소스 디자인 시스템 Primer React 설치), 클로드 코드에서 `/design-sync` 실행 → 클로드 디자인에 레포가 동기화 → 프로토타입 생성 시 해당 디자인 시스템 선택.
- 클로드 디자인이 작업 착수 전 "이 화면의 핵심은?", "정산 기능을 넣을지", "시안 몇 개를 볼지" 등 기획/디자인 질문을 먼저 던지는 점을 강점으로 평가(클로드 코드는 이런 질문을 빠뜨리는 경우가 많음).
- `/design-sync`가 안 보이면 터미널에서 클로드 코드를 최신 버전으로 업데이트해야 한다.

## 언급된 도구/프레임워크

- `Claude Design (클로드 디자인)` — 유료 회원 대상, 클로드 웹사이트의 '디자인' 메뉴에서 사용. 프레젠테이션·슬라이드·프로토타입·와이어프레임·문서(PDF)·애니메이션 생성.
- `Design-sync (/design-sync)` — 클로드 코드 코드베이스의 디자인 시스템을 클로드 디자인으로 동기화하는 신규 슬래시 명령.
- `Claude Code` — 디자인 시스템의 원천 역할. 최신 버전 필요.
- `Primer React` — GitHub이 만든 오픈소스 디자인 시스템(컴포넌트·아이콘·디자인 토큰 완비), 전방위 테스트용으로 사용.
- `Vercel` — 클로드 디자인 결과물의 퍼블릭 원클릭 배포 통합.
- `Figma / Zeplin` — 기존 코드-디자인 일관성 도구(비교 대상). 결국 사람이 QA로 직접 확인해야 했던 한계.
- `MCP / 커넥터` — 클로드 디자인에서도 사용 가능하며 클로드·클로드 코드와 동기화됨.

## 트랜스크립트 발췌

> 클로트 디자인에는 몇 가지 아쉬운 부분들이 있었습니다. 하나는 토큰을 너무 많이 소진한다는 거였어요. 두 번째는 디자인 에디터 기능이 약하다는 거였어요. 그리고 세 번째, 저는 이게 사실 큰 부분인데요. 디자인 시스템이 클로드 코드와 클로드 디자인 사이에서 동기화가 되지 않는다는 거였어요. (…) 이번 클로드 디자인 업데이트를 통해서 이제 클로드 코드의 코드베이스에서 디자인 시스템을 통째로 클로드 디자인으로 가져올 수 있게 됐어요. 그게 무슨 말이냐면 클로드 코드랑 클로드 디자인 사이에서 디자인 시스템의 동기화가 가능해졌다는 거예요. (…) 결론적으로 디자인이 구현되는 단계에서 항상 달라지고 일관성이 떨어지는 문제를 이번 디자인 업데이트에서 그 해법을 제시했다는 거예요. 그리고 에디터를 통해서는 디테일한 크기를 조정하고 위치를 변경하는 부분들을 상세하게 그 자리에서 바로바로 할 수 있게 됐습니다. 프롬프트를 쓰지 않더라도 할 수 있는 거예요. (…) 토큰적인 부분에서는 앤트로픽이 공식 사이트에서도 사용량이 훨씬 더 늘어났다고 하고 있는데요. 기존에는 클로드 디자인만 따로 분류되어서 토큰 사용량 제한이 있었어요. 근데 클로드 코워크, 클로드 코드 이런 것과 다 통합돼서 토큰 사용량을 공유할 수 있게 되었습니다. (…) 이번에 Vercel 같은 걸 통합해서 바로 퍼블릭 웹에서도 볼 수 있게 되었습니다. (…) 이 버튼을 누르게 되면 슬래시 디자인 싱크가 나옵니다. 이 디자인 싱크 기능은 클로드 디자인이 아니라 클로드 코드에서부터 시작할 수 있어요. 이거는 클로드 코드가 디자인 시스템의 원천이라고 보시면 되고요. 그게 클로드 디자인으로 동기화돼서 올라가는 거다, 이렇게 이해하시면 될 것 같아요.
