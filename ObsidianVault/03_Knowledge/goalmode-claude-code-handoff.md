# 클로드 코드 넘김 명령 (골모드 작동)

> 사용법: `obsidian-agent-brain-system` 폴더에서 Claude Code 실행 후, 아래 ``` 블록 전체를 복사해 붙여넣기.
> 골모드가 기동되어 갭 분석에서 "바로 적용 가능"으로 분류된 3개 항목을 계획→승인→구현한다.

---

```
GOAL MODE 기동.

# 입력
GOAL: 이 vault를 정적 노트 저장소에서 "능동 브레인"으로 업그레이드한다.
      구체적으로 (1) 영구 기억 구조, (2) 노트 자동 주입, (3) 목표 위임 루프(골모드) 3개를 구현한다.
DEADLINE: 없음
CONSTRAINTS:
  - 단일 책임 / named export만 / any·unknown 금지
  - 기존 Obsidian vault를 백엔드로 유지(버리지 말 것). 에이전트 레이어만 추가.
  - 단순성 우선: 복잡한 방법 택할 때 "왜 단순한 방법이 안 되는가" 한 줄 명시.
  - 화면 감시·비즈니스 자동화는 범위 밖. 구현하지 말 것.

# 작동 규칙
[1] 분해: GOAL을 측정 가능한 완료 조건(DoD) 1문장으로 재정의하고 3~7단계로 분해.
    각 단계에 산출물 명시. 불명확한 전제는 실행 전 1회만 묶어서 질문.

[2] 승인 게이트: 분해 결과를 다음 형식으로 출력하고 멈춘다.
    구현 방식 / 수정·생성 파일 / 단계별 Todo / 트레이드오프
    파일 수정·코드 생성은 승인 전 절대 금지. 읽기·분석은 즉시 가능.
    마지막 문구: "계획이 완료되었습니다. 검토 후 승인해주세요. 아직 코드를 수정하지 않았습니다."

[3] 실행: 승인 후 단계 순서대로. 완료마다 한 줄 보고. 막히면 즉시 중단+원인+대안2개.
    같은 오류 2회 시 자동 재시도 금지, 에스컬레이션.

[4] 기억 기록: 완료 시 goals/{YYYY-MM-DD}-{슬러그}.md 에 목표·결과·배운점 기록,
    관련 노트에 [[백링크]] 연결. 실패 접근법은 "다시 쓰지 말 것" 태그와 함께 기록.

[5] 패턴 감지: 기록 시 goals/ 스캔 → 3회+반복 작업=자동화 후보 제안,
    반복 실패 단계=프로세스 결함 보고, 유사 목표=과거 결과 링크. 제안만, 구현은 별도 승인.

[6] 종료: DoD 충족 시 "GOAL COMPLETE"+요약+기록경로 출력 후 골모드 해제.
    "골모드 중지" 입력 시 진행상태 기록 후 즉시 해제.

# 금지
승인 없는 실코드 생성·외부 호출 / 스코프 크리프 / 시크릿·API키·PII 취급 시 즉시 중단 고지 /
결제·인증·보안 단계는 규모 무관 항상 개별 승인.

지금 [1] 분해부터 시작하고 [2] 승인 게이트에서 멈춰라.
```

---

## 붙여넣기 전 확인
- Claude Code를 **vault 루트**(`obsidian-agent-brain-system`)에서 실행할 것. 하위 폴더에서 실행하면 참고 자료 경로가 어긋남.
- 골모드는 [2]에서 반드시 멈추고 승인을 기다림 → 재하님의 "승인 전 실코드 금지" 원칙 준수.
