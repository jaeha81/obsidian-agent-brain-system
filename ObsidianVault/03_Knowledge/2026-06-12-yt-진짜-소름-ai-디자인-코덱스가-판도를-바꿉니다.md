## 영상 핵심 요약 (watch 분석 — 2026-06-12)

영상은 두 가지 AI 디자인 도구를 소개한다.

### 도구 1: Design Lang CLI
- **GitHub**: `https://github.com/Manavarya09/design-extract.git`
- **기능**: 웹사이트 URL 하나 입력 → Design Token + Figma 변수 + Tailwind 설정 + DESIGN.md 자동 추출
- **핵심**: Claude Code / Codex가 이해할 수 있는 **자동 통역기** — 디자이너 언어(Figma) ↔ 개발자·AI 언어 동시 출력
- **설치 방법**: Codex에 GitHub URL 붙여넣기 → "설치해줘" 한 마디

### 도구 2: OpenAI Codex Product Design Plugin
- **위치**: Codex 플러그인 마켓 → "Product Design" 활성화
- **기능**:
  - URL 입력 → 웹사이트 클론 (3D 모션, 마우스 인터랙션, 스크롤 포함)
  - Three.js 기반 3D 렌더링 자동 구현
  - 원본과 클론을 스스로 비교하며 싱크로율 향상
  - 색상·폰트·레이아웃 수정 요청 실시간 반영
- **한계**: 이미지는 코드 구현 불가 (GPT 이미지 생성으로 대체), 실제 백엔드 기능은 별도

### 두 도구의 조합 워크플로우
```
레퍼런스 URL
  └─ Design Lang CLI → DESIGN.md (구조/토큰/변수)
       └─ Codex Product Design Plugin → 실제 웹사이트 구현 (3D/인터랙션 포함)
            └─ 수정 요청 → 자동 비교·조정 → 완성
```

## Bucky Agent 능동 디자인 적용 플랜

### 현재 가능한 자동화 (Design Lang CLI 기반)

Bucky가 웹 UI 구현 요청을 받을 때:

```
사용자: "이 사이트 스타일로 만들어줘 [URL]"
  └─ Bucky → Claude Code 패킷 생성:
       {
         tool: "design-lang",
         input: "[URL]",
         output: ["design-tokens.md", "tailwind.config.js", "DESIGN.md"],
         next: "Claude Code가 DESIGN.md를 Context로 사용해 구현"
       }
```

### Design Lang 설치 방법
- Codex에게: "이 레포 설치해줘 → https://github.com/Manavarya09/design-extract.git"
- 또는 터미널: 공식 설치 명령어 직접 실행

### Product Design Plugin 사용 시나리오 (수동, Codex 앱)
1. OpenAI Codex 앱 실행
2. 좌측 사이드패널 → 플러그인 → Product Design → + 버튼 설치
3. 새 채팅 열기 → 좌측 + → 플러그인 → Product Design 선택
4. 채팅창 상단 분홍색 "프로덕트 디자인" 표시 확인
5. 레퍼런스 URL 입력 + "그대로 구현해줘" 프롬프트

### Bucky가 사용자에게 알려줄 라우팅 규칙

| 요청 유형 | 도구 | 에이전트 |
|-----------|------|---------|
| 웹사이트 스타일 분석·토큰 추출 | Design Lang CLI | Claude Code 또는 Codex |
| 정적 이미지 → 인터랙티브 프로토타입 | Product Design Plugin | OpenAI Codex 앱 (수동) |
| 3D 모션 그래픽 구현 | Product Design Plugin | OpenAI Codex 앱 (수동) |
| 토큰 기반 일관된 스타일 구현 | Design Lang + Claude Code | Claude Code |
