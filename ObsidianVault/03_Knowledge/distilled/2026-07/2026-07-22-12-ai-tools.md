> AI 도구 사용 원칙 — Claude Code Max 운영 현황, 버전 안전/위험 목록(v2.1.34 안전, v2.1.69~91 캐시 버그), AI 응답 금지/필수 규칙, Hermes 보류·Paperclip 벤치마크 결과, 6-질문 사전 체크리스트.

## 핵심 인사이트

- **Claude Code 운영**: Max 플랜, 주력 모델 claude-opus-4 / claude-sonnet-4. 터미널 직접 실행 + tmux 멀티에이전트. 로컬 LLM 없음 — 완전 API 기반.
- **버전 관리 (Critical)**: v2.1.34 = 안전 확인됨. **v2.1.69~v2.1.91 = 캐시 비효율 버그** — `--continue -p` / `--resume -p` 플래그 사용 시 캐시 효율 급감 → 반복 작업 토큰 비용 급증. 모니터링 명령: `uvx -q ccctail -f`.
- **AI 응답 금지 4항**: 아부(칭찬 먼저·문제 나중), 이미 설명한 내용 재질문, 묻지 않은 확장성/마이크로서비스 제안, 승인 전 실코드 생성.
- **AI 응답 필수 4항**: 약점·리스크 반드시 언급 / 전제 오류 시 "⚠️ 전제 검토: [문제]. 대안: [해결책]. 계속 진행할까요?" / 복잡성 추가 시 "왜 단순한 방법이 안 되는가" 한 줄 명시 / 스크린샷 제공 시 즉시 최우선 참고.
- **Hermes Agent**: 벤치마크 완료, 도입 보류. 스킬 자동 추출·컨텍스트 압축·세션 검색 모두 현재 규모에서 불필요. 재검토 시점 = 동시 진행 프로젝트 5개 이상.
- **Paperclip**: JH-EstimateAI 벤치마킹에 활용 → 4대 업그레이드 우선순위 도출: ① Goal Tree View ② Per-Agent Token Budget Tracking ③ Cron Trigger Layer ④ EstimateAI Template Packaging.
- **6-질문 사전 프로젝트 체크리스트**: 목적 명확? / 스택 확정? / 보안 요구사항? / 배포 환경 결정? / 테스트 범위? / 납품 형태 명확?
- **보안 규칙**: `.env`·API 키·PII 채팅 붙여넣기 절대 금지, 외부 API 연동 시 인증정보 전달 방식 검토 필수, 보안·결제·인증 작업은 규모 무관 항상 계획→승인.
- **비용 관리**: Max 플랜 토큰 예산 관리, 버그 버전 회피로 캐시 비용 절감, Haiku는 IntentGate 등 빠른 분류에만 사용(비용 최적화). 프로젝트별 특이사항은 각 레포 CLAUDE.md에 기록 — 전역 vs 로컬 규칙 계층 구조.
