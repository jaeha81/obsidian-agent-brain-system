## 핵심 인사이트

- Obsidian = 단순 메모장이 아닌 세컨드 브레인. 저장 대상: 개발 기록, 사업 아이디어, 인테리어 지식, 투자 기록, 자동화 시스템 설계, 프롬프트, 하네스 구조, 프로젝트별 결정사항, AI 대화 핵심, 장기 지식 자산(GitHub 연동).
- 초기 설계 Vault 구조 = `JH_Second_Brain/` 11개 폴더: 00_Inbox / 01_Profile(재하_백과사전·핵심목표·운영원칙) / 02_Business(JH_Connect·Space819·도연인테리어·더나은공간) / 03_Interior(견적시스템·시공관리·감리·자재·협력사·하자대응) / 04_AI_Automation(Make·n8n·GAS·Webhook·OpenRouter) / 05_Development / 06_Harness(JH_Harness·Agent_Packs·Supervisor_Agent) / 07_Investment(주식·가상화폐·선물옵션·BitMEX) / 08_Content(YouTube·Shorts·Blog) / 09_SecondBrain(LLM_Wiki·GitHub_Sync·Checkpoints) / 10_Archive. **현행 OABS 볼트(00_System·01_RAW·03_Knowledge 3층)와 다른 구세대 설계 — 역사 기록으로 보존.**
- **"체크포인트"** = 재하가 부르는 대화 세션/프로젝트 진행 단위. 역할: 목표 확인, WIP 정리, 막힌 부분 기록, Next 정리, KPI 확인, 세션이 바뀌어도 본질을 잃지 않게 하는 기준점 — 현재 HANDOFF_LOG/세션 인계 체계의 전신.
- 체크포인트 시작 헤더 5요소: Top-1 목표 / WIP 2개 이하 / KPI 3개 / 금지 3개 / Next 3개. 종료 로그 4요소: Done 3 / Blocked 3(원인 명시) / Next 3 / KPI 스냅샷.
- GitHub Sync: `jaeha81` 저장소 = 코드+기록, Obsidian Vault = markdown 지식 저장소. 연동은 Obsidian Git 플러그인 또는 수동 push.
- 세컨드 브레인 업데이트 트리거 11종: 프로젝트 시작/완료, 브랜드명·기술스택·수익화모델·법적기준·개발 운영방식 변경, 중요 의사결정, 체크포인트 종료 후 핵심 누적, 실패 프로젝트 원인 기록, 성공 자동화 흐름 기록.
- AIMY 협업 역할 분담: 재하=방향 제시/판단/컨펌/사업 의사결정, AIMY(GPT)=전체 계획/구조화/프롬프트/검증 기준, Claude Code=구현, Codex=코드 검증/오류 탐지, Obsidian=지식, GitHub=코드+기록, Google Sheets/DB=업무 데이터, Netlify/Vercel=배포, Make/n8n/GAS=자동화.
- AIMY 답변 선호: 한국어, 초보자 설명 반복 금지, 실행형 결과물 우선, 프롬프트는 복사 가능한 단일 블록, Claude Code용/Codex용 구분, 꼭 필요한 컨펌만 짧게.
