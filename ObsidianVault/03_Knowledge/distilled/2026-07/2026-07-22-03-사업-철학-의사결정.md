> 재하의 사업 철학과 의사결정 프레임 — "AI=협업 파트너, 수익은 시스템이 작동한 결과", 현실 필터 10문, 의사결정 14기준, 운영 헌장 10조.

## 핵심 인사이트

- 철학 전환: 초기 "AI=돈벌이 도구" → 현재 "AI=협업 파트너, 돈은 시스템이 제대로 작동한 결과로 따라온다".
- 핵심 철학 7: 실행 가능한 구조 먼저 / 법적·보안·저작권 회피 중시 / 무료·저비용 시작 후 점진 확장 / 자동화는 반드시 현실 업무와 연결 / 범용 AI보다 인테리어 특화 AI.
- **현실 필터 10문** — 모든 신규 프로젝트가 통과해야 함: 실제 구현 가능? 저비용 시작 가능? 법적 문제 없나? API 비용 감당 가능? 실사용 이유 있나? 현장에 실질 도움? 수익화 가능? 유지보수 가능? 데이터·보안 고려? MVP로 검증 가능?
- 의사결정 14기준 중 특징적인 것: "데이터가 쌓이는가", "AI가 추천하기 좋은 구조인가", "나중에 상품화할 수 있는가", "Claude Code/Codex가 구현 가능한 수준인가".
- 역할 분담: 재하=방향 제시·판단·컨펌·사업 결정, AI=그 구조 안에서 실행. "구현해"=승인 키워드. 불확실한 규모는 '중'으로 처리(계획→승인).
- 운영 헌장 요체: Claude Code가 구현, Codex가 검증, AIMY가 전략·프롬프트·검증 기준 담당. 모든 중요 결정은 Obsidian에 기록. 모든 웹앱은 모바일+데스크탑 고려, 모든 시스템은 수익화 가능성 고려.
