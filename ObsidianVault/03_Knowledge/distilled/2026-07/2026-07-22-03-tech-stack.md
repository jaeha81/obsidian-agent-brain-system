> 재하의 표준 기술 스택(FastAPI + Next.js 14 + Supabase + Claude API)과 필수 코딩 규칙, 에이전트 아키텍처 원칙, 실전 버그 패턴을 정리.

## 핵심 인사이트

- 표준 스택: Backend=FastAPI(Python 비동기), Frontend=Next.js 14(App Router, Server Components 우선), DB=Supabase(PostgreSQL+Realtime+RLS+Auth), AI=Claude API, 인프라=Vercel(FE)/Railway·VPS(BE). 에이전트 프레임워크는 LangGraph 참고하되 자체 구현.
- 필수 코딩 규칙 5: ① named export만(`export default` 금지) ② `any`/`unknown` 타입 금지 ③ 파일당 단일 책임 ④ 중복 엔드포인트 금지 ⑤ 결제 로직엔 항상 사용자 확인 단계.
- 모델 선택 기준: 복잡한 판단/오케스트레이터=opus, 일반 에이전트=sonnet, 빠른 분류(IntentGate)=haiku (비용 최적화).
- **핵심 버그 패턴**: async 함수 안에서 sync `Anthropic()` 클라이언트 사용 → FastAPI 이벤트루프 블로킹. `AsyncAnthropic()` + `await`가 정답 (JH-키아누에서 실제 발견된 사례).
- 에이전트 아키텍처 원칙: 판단(Supervisor)과 실행 분리, 파일 기반 로그로는 런타임 동작 강제 불가, YAML 스키마도 마찬가지 → 강제는 코드로 구현해야 함. 승인 패턴="구현해" 키워드 매칭, 단일 human intervention point.
- UI: Tailwind CSS 주력 + shadcn/ui + Three.js(3D 시각화). `"use client"` 최소화.
- 테스트 전략: 통합 테스트 우선, 유닛 테스트 후순위 — PreCon AI 6/6 통합 테스트 통과 기준, JH-견적은 실제 Excel 데이터로 정규화 검증.
