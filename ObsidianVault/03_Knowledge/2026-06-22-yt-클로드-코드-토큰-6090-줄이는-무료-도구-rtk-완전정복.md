# 클로드 코드 토큰 60~90% 줄이는 무료 도구 rtk 완전정복

![thumbnail](https://i.ytimg.com/vi/B114J2sQxco/maxresdefault.jpg)

## 요약
AI 코딩 도구 요금의 상당 부분이 "눈으로 한 번도 안 보고 버리는 터미널 출력"에서 새어나간다는 문제에서 출발한다. rtk(Rust Token Killer)는 명령 결과가 AI에게 전달되기 직전 길목을 지키며 군더더기를 걷어내고 핵심만 남겨 넘기는 무료 오픈소스(Apache 2.0) 실행 파일이다. 출력을 줄이는 4가지 방법(잡음 제거·묶기·잘라내기·중복 접기)과 공식 절감표(30분 코딩 세션 전체 약 80% 절감), 한 줄 설치 + 한 줄 Claude Code 훅 연결, 절감 확인 명령(rtk gains/discover)까지 다룬다. 끝으로 rtk는 터미널 출력만 담당하고 파일 읽기(Read)는 손대지 못하므로, 그 공백은 Serena가 메운다는 역할 분담을 짚는다.

## 핵심 개념
- **토큰이 새는 지점**: AI는 테스트·git·로그 등 명령을 수없이 실행하고 각 결과 수백 줄을 전부 토큰으로 "사서" 읽는다. 정작 필요한 건 통과/실패 몇 줄뿐. 다음 작업 때 그 긴 출력을 다시 떠올리며 또 토큰을 쓴다.
- **rtk = 문지기(길목) 도구**: 명령 결과가 AI로 들어오는 길목에서 잘라낸다. Serena가 "읽기 전 단계"라면 rtk는 "결과가 들어오는 길목".
- **4가지 절감 방법**: ① 잡음 제거(빈 줄·장식 기호) ② 묶기(비슷한 항목을 하나로) ③ 잘라내기(중요 부분만, 반복 뒷부분 제거) ④ 중복 접기(같은 로그 100번 → "100번 반복" 한 줄).
- **공식 절감표(30분 세션 기준)**: 파일 목록 80%, 검색 80%, 변경 내역 확인(diff) 75%, 테스트 실행 90% 절감. 합계 약 11만8천 토큰 → 약 4만 토큰, 전체 약 80% 절감.
- **설치·연결이 각 한 줄**: 터미널에 한 줄로 설치, `rtk init`류 명령 한 줄로 Claude Code 훅 자동 설정(기존 설정은 백업 생성). 이후 프롬프트·명령을 바꿀 필요 없고 절감 과정 자체엔 추가 토큰이 안 든다.
- **이름 충돌 주의**: 동명의 다른 도구가 있음 → 설치 후 `rtk gains`로 절감 통계가 뜨면 정상, 명령을 못 찾으면 다른 도구가 깔린 것.
- **확인 명령**: `rtk gains`(누적 절감 통계·일자별 그래프), `rtk discover`(과거 작업 기록을 훑어 rtk를 안 거쳐 놓친 절감 기회 탐지).
- **rtk의 한계 + Serena와의 분업**: rtk는 터미널 명령 출력만 줄인다. Claude Code가 파일을 읽을 때는 내부 Read 기능을 써서 rtk를 거치지 않는다 → 코드 통째 읽기 토큰은 Serena가 담당(고칠 함수만 콕 집어 읽기). 둘은 경쟁이 아니라 서로 다른 구멍을 막는 분업 관계(그래서 Headroom이 둘을 묶어 설치).
- **주의점**: 절감률은 작업 종류마다 편차가 크고, 출력을 줄여 보여주므로 가끔 전체 출력을 봐야 할 때도 있다.

## 언급된 도구/프레임워크
- `rtk (Rust Token Killer)` — Rust로 만든 터미널 출력 토큰 절감 도구. GitHub 스타 6만+, Apache 2.0, 단일 실행 파일. 60~90% 절감.
- `Serena` — AI가 코드를 통째로 읽지 않고 고칠 함수만 집어 읽게 하는 "읽기 전 단계" 절감 도구. rtk가 못 막는 파일 읽기 구멍 담당.
- `Headroom` — rtk와 Serena를 한 번에 묶어 설치해주는 도구.
- `Claude Code` — rtk 훅 연결 대상. 명령 실행마다 rtk를 자동 경유.
