## 에이전트 활용 지침

### 언제 이 카탈로그를 참조하나

- 사용자가 특정 기능/프로젝트를 언급할 때 → 해당 레포 식별
- 새 개발 작업 착수 시 → 기존 유사 레포 중복 여부 확인
- 배포·디버그 요청 시 → 레포 언어/상태 파악 후 적절한 스킬 선택
- 시스템 인프라 작업 시 → 🏗️ 시스템/브레인 카테고리 우선 확인

### 레포 → 작업 라우팅 규칙

```
레포 분류        → 권장 스킬/행동
─────────────────────────────────
🏗️ 시스템/브레인 → jh-github, jh-debug, jh-deploy
🔧 개발 프로젝트  → jh-code-review, jh-deploy, jh-qa-release-checker
📦 아카이브       → 수정 전 사용자 확인 필수
```

## 카탈로그 갱신 지침

### 자동 갱신 (권장)

```bash
# 프로젝트 루트에서 실행
python scripts/github_repo_cataloger.py

# 토큰 있을 때 (Private 레포 포함, Rate limit 없음)
GITHUB_TOKEN=<token> python scripts/github_repo_cataloger.py

# 미리보기만 (파일 변경 없음)
python scripts/github_repo_cataloger.py --dry-run
```

**출력 파일**:
- `ObsidianVault/03_Projects/github-overview.md` — 전체 대시보드
- `ObsidianVault/03_Projects/github-repos/*.md` — 개별 레포 노트
- `ObsidianVault/03_Knowledge/github-catalog.md` — 카탈로그 원본 (수동 또는 다른 에이전트가 생성)

### 이 파일 수동 갱신 시점

- 레포 10개 이상 추가/삭제된 경우
- 레포 분류가 변경된 경우 (예: 프로젝트 → 시스템)
- 새로운 핵심 레포가 생겨 빠른 식별이 필요한 경우
