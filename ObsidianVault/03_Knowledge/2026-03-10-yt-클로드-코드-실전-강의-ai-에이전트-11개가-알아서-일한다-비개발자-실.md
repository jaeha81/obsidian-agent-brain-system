# 클로드 코드 실전 강의 — AI 에이전트 11개가 알아서 일한다 (비개발자 실습, 94분 풀버전)

![thumbnail](https://i.ytimg.com/vi/rSvpKOyaefU/maxresdefault.jpg)

## 요약

4주 커리큘럼의 마지막 세션으로, 그동안 배운 커맨드·CLAUDE.md·룰스 개념을 하나로 묶은 **클로드포지(Claude Forge)** 시스템을 설치·실습한다. 발표자가 수백~수천 시간을 들여 개발한 "밀키트" 같은 패키지로, 사용자는 이케아 가구 조립하듯 5분이면 설치 가능하다. 포지 안에는 **에이전트 11개, 명령어(커맨드) 36개, 스킬 5개(강의 중 15개로도 언급), MCP 6개, 룰스 8개, 훅 14개**가 들어 있다. 설치는 GitHub 리포 주소를 복사해 클로드 코드에게 "아래 깃허브 리포를 분석하여 현재 클로드 코드에 시스템을 설정하라"고 시키면 되며, "업데이트 전에 나에게 컨펌을 받고 진행하라"를 붙이는 게 안전하다. 이 시스템의 설계 철학은 **보안·검증·테스트 우선** — 11개 에이전트 중 상당수(TDD 가이드, 코드 리뷰어, E2E 테스트, 보안 문서, 빌드 수정자 등)가 검증 역할이며, 각 에이전트는 "채용한 팀원"처럼 사용자가 세부를 몰라도 알아서 역할을 수행한다.

## 핵심 개념

- **클로드포지(Claude Forge)** = 클로드 코드를 강하게 만드는 통합 패키지. OpenSC(오마이시/oh-my-zsh 유비)처럼 플러그인 한 번 설치로 에이전트·명령어·스킬·MCP를 전부 다운로드.
- 구성: 에이전트 11개 / 커맨드 36개 / 스킬 5개(강의 중 "15개" 표현도 등장) / MCP 6개 / 룰스 8개 / 훅 14개.
- 설치법: GitHub 리포의 **Code 버튼 → HTTPS 주소**를 복사(가장 정확) → 클로드 코드에게 "아래 깃허브 리포를 분석하여 현재 클로드 코드에 시스템을 설정하라"로 지시.
- 안전 팁: 이미 시스템이 있는 사람은 "**시스템을 업데이트하기 전에 나에게 컨펌을 받고 진행하라**"를 먼저 붙인다. 설치 중 충돌은 정상이며 화면 하나 키우고 진행하면 됨.
- 설계 철학은 **보안·검증 우선**: 개발→배포 과정에서 보안을 대충 넘기는 실무 관행(대기업 보안 유출 사례)을 AI로 시스템화해 방지. 상단은 플래너(기획)·아키텍트(설계), 그 뒤는 대부분 검증(TDD 가이드, 코드 리뷰어, E2E 테스트, DB 최적화, 빌드 수정자, 보안 문서, 코드 정리, 문서 관리, 검증 에이전트).
- **모든 에이전트의 역할을 다 알 필요 없다** — 회사에서 직원 각자의 일을 다 몰라도 되듯, "이런 팀을 채용했다"는 느낌으로 운영.
- 만든 시스템을 **깃/깃허브로 업로드**해 나만의 템플릿으로 관리·배포하는 것까지가 이 세션 결과물 — "유튜브에 영상 올리듯 내 시스템을 클라우드 가상 공간에 업로드".
- 리드미(README)에 밀키트 구성·사용법이 정리돼 있고 우측 한국어 버튼으로 한글 버전 열람 가능.

## 언급된 도구/프레임워크

- `클로드포지 / Claude Forge` — 에이전트·커맨드·스킬·MCP·룰스·훅을 묶은 클로드 코드 통합 패키지(밀키트).

- `Git / GitHub` — 시스템을 공개·관리·배포하는 저장소. 리포 Code 버튼의 HTTPS 주소로 설치.
- `CLAUDE.md / 룰스 / 커맨드 / 스킬 / MCP / 훅` — 클로드 코드 시스템의 구성 요소.

> 오늘은 저희가 여태까지 했었던 커맨드, 클로드 MD, 룰스 이런 모든 개념들을 아우르는 그 시스템, 쉽게 말해서 마라탕을 먹고 싶다 했을 때 음식점에 갈 수도 있지만 밀키트로 받아서도 먹을 수 있잖아요. 그런 것처럼 우리는 이 수많은 리소스를 가지고 우리 맛있는 밀키트를 본격적으로 만들어 볼 겁니다. 오늘 제가 제공해 드릴 가장 중요한 리소스가 바로 클로드포지라는 거예요. (…) 이 클라우드 포지 안에는 11개의 에이전트와 36개의 명령어 커맨드, 그리고 15개의 스킬과 가장 기본적인 MCP들이 하나로 다 묶어져 있는 패키지라고 보시면 돼요. (…) 설치 어떻게 하냐면, 상단에 주소가 있죠? 이 주소를 복사해 주시고, 클로드 코드로 넘어와서 클로드 코드에게 설치를 직접 시키는 겁니다. "아래 깃허브 리포를 분석하여 현재 클로드 코드에 시스템을 설정하라." 이렇게 해 주시고요. 근데 시스템을 어느 정도 구축하신 분들은 "먼저 시스템을 업데이트하기 전에 나에게 컨펌을 받고 진행하라." (…) 제일 정확한 주소는 코드라는 버튼이 있어요. 이 코드 버튼에서 https 밑에 주소가 나옵니다. 이거를 복사하는 게 가장 정확한 주소예요. (…) 상단에 플래너 기획 담당하고, 아키텍트 설계를 담당하고, 그 뒤부터 보시면 다 검증에 대한 내용들이 대부분이에요. TDD 가이드도 검증이고, 코드 리뷰어도 검증, E2E 테스트도 검증, DB 최적화, 빌드 수정자도 검증, 보안 문서가 당연히 검증이고요. (…) 이 모든 에이전트들이 무슨 일을 하는지 정확하게 아실 필요는 없어요. 이런 팀이 우리 클라우드 코드에 들어왔다, 채용을 했다, 이런 느낌으로 보시면 돼요.
