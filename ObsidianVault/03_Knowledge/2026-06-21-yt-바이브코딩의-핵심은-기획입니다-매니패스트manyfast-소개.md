# 바이브코딩의 핵심은 기획입니다 | 매니패스트(ManyFast) 소개

![thumbnail](https://i.ytimg.com/vi/KK3atit8Ww4/maxresdefault.jpg)

## 요약
바이브 코딩에서 가장 중요한 단계는 코드가 아니라 기획이라는 주장을 중심으로, 기획 전용 도구 ManyFast(매니패스트)를 실제 사용 화면으로 시연한다. 마크다운 문서는 AI가 다루기 좋지만 텍스트 나열이라 사용자 흐름·중요도·누락 지점을 파악하기 어렵다는 한계를 지적하고, 그 대안으로 "화이트보딩"처럼 기획을 시각화하는 접근을 제시한다. ManyFast는 AI와의 문답을 통해 PRD → 기능 명세서(트리 구조) → 유저 플로우(사이트맵) → 와이어프레임까지 순차적으로 구체화하며, 각 단계의 변경은 승인/거절 게이트를 거친다. 마지막으로 산출 문서를 파일로 내보내거나 MCP로 Claude Code 같은 코딩 에이전트에 직접 연동하는 방법까지 보여준다.

## 핵심 개념
- **기획이 바이브코딩의 병목**: 단순해 보이는 서비스도 기획하면 기능이 폭증한다. 구조를 머릿속에 넣지 않고 개발을 시작하면 프로젝트가 통제를 벗어난다.
- **마크다운 PRD의 한계**: 텍스트 관리라 중요 부분 파악이 어렵고, 복잡한 그래프·테이블·사용자 흐름 표현이 취약하다 → "시각화"가 해법.
- **화이트보딩 기법**: 머리로만 고민하지 않고 칠판에 문서화해 눈으로 보면 무엇이 중요한지·무엇이 빠졌는지 한눈에 보인다. 브레인스토밍에 최적화.
- **4단계 산출물 파이프라인**: ① PRD(목표·사용자·문제·해결·차별점·시나리오·핵심지표) → ② 기능 명세서(대표 기능을 트리로, 하위 기능을 AI가 확장) → ③ 유저 플로우(로그인→대시보드→메뉴→세부 페이지를 사이트맵처럼) → ④ 와이어프레임.
- **AI 문답으로 구체화**: 프롬프트 제출 후 마스코트 "매니"가 되묻는 질문에 답하며 기획을 점진적으로 구체화. 만족할 때까지 추가 질문 가능.
- **승인/거절 게이트**: AI가 임의 생성한 기능은 검토 후 거절 버튼으로 제거, 일괄 승인도 가능. 문서 간 정합성(PRD↔기능명세)이 자동 유지돼 "문서가 따로 노는" 문제를 방지.
- **와이어프레임이 하이라이트**: 탄탄한 기획 문서가 받쳐주니 Claude Design·Figma 같은 전용 도구보다 완성도가 높을 때가 많았다. 기획 단계에선 화려한 폰트·색보다 "정보 배치"가 핵심이라 밋밋한 레이아웃이 오히려 검토에 유리.
- **모델 선택 가능**: 반영 계획 브리핑에 Claude Opus 등 모델을 지정할 수 있다.

## 언급된 도구/프레임워크
- `ManyFast (매니패스트)` — 바이브코딩 기획 전용 도구. AI 문답으로 PRD·기능명세·유저플로우·와이어프레임 생성, MCP로 코딩 에이전트 연동.
- `MCP` — ManyFast 산출 문서를 Claude Code 등 에이전트에 직접 연동하는 프로토콜.
- `Claude Code` — MCP 연동 대상 코딩 에이전트(터미널에 연동 명령 붙여넣기).
- `Claude Opus` — 변경 반영 계획에 지정 가능한 모델.
- `Figma / Claude Design` — 비교 대상 디자인 도구(와이어프레임 품질 비교).

## 트랜스크립트 발췌
> 오늘은 바이브 코딩에서 가장 중요한 기획 얘기를 해 볼 거예요. 바이브 코딩 하다 보면 마크다운 문서를 많이 쓰게 되는데요. 가벼워서 AI가 쓰기 좋은 형식이지만 단점도 많습니다. 왼쪽에 보이는 이미지가 마크다운으로 작성된 PID인데요. 보시다시피 내용이 텍스트로만 관리돼서 중요한 부분을 파악하기 어렵습니다. 또 복잡한 그래프나 테이블도 표현하기 어렵고요. 특히 기획 단계에서는 사용자 흐름을 파악하는게 중요한데 마크다운만으로는 잘 검토하는데 한계가 있습니다. 이때 필요한게 바로 시각화입니다. 회의할 때 칠판의 생각 정리해 본 경험 다들 있으실 텐데요. 아무래도 머리로만 고민하는 것보단 칠판 위에 문서화해 놓고 그걸 다시 눈으로 읽다 보면 뭐가 중요한지 한 눈에 보이거든요. 또 빠진 부분도 명확히 알 수 있고요. 이런 기법을 화이트 보딩이라고 합니다 (…) 바로 매니페스트입니다. 매니페스트는 바이브 코딩의 기획 과정을 도와주는 도구입니다. AI랑 대화하면서 PRD 기능 명세서 유저플로우 그리고 와이어프레임까지 만들 수 있고요. MCP로 코딩 에이전트랑 연동할 수도 있어요. 특히 와이어 프레임 기능이 좀 재밌는데요. 기획을 디테일하게 잡고 나서 와이어 프레임까지 만드는 거라 디자인 퀄리티가 웬만한 디자인 툴들보다 좋습니다 (…) 사실 단순해 보이는 서비스여도 막상 기획하고 나면 이렇게 기능이 엄청 많아지는 경우가 많습니다. 그래서 무작정 개발을 시작하면 프로젝트가 너무 거대해지고 내 통제를 벗어나는 경우가 많아요. 그전에 미리 잘 검토해서 프로젝트 구조가 내 머릿속에 잘 입력된 채로 진행하는게 좋겠죠 (…) 유저 플로우는 사용자 여정을 시각화하는 문서입니다 (…) 이렇게 사이트 맵처럼 정리돼서 시각적으로 검토하기가 되게 편합니다 (…) 클로드 디자인이나 피그마 같은 디자인 전용 도구들보다도 매니패스트의 와이어 프레임이 더 완성도가 높을 때가 많았거든요. 이유를 생각해 보면 사실 디자인도 크게 보면 기획의 한 부분이라서 기반이 되는 기획 문서들이 탄탄하게 받쳐 주니까 디자인 완성도 자연스럽게 좋게 나온 것 같아요.
