# 화제의 하네스 LazyCodex 사용법 완전 정리 (트위터, 쿠팡 코덱스로 검색 가능)

![thumbnail](https://i.ytimg.com/vi/B8Esyo5JveM/maxresdefault.jpg)

## 요약

쓰레드에서 화제인 **LazyCodex** 하네스를 소개한다. OpenCode를 만든 연규 님이 코덱스(Codex) 위에서도 OpenCode 같은 경험을 낼 수 있게 제작한 하네스로, 한 줄 설치가 가능하고 코덱스 기반이라 **윈도우에서도 정상 동작**한다. 핵심은 `ULW 루프`(작업 완수까지 자동 반복), `ULW 플랜`(프로메테우스/메티스 플래너가 아이디어를 실행 가능한 작업으로 변환), `start work`(계획 실행)의 세 명령이며, 이날 새로 탑재된 **울트라서치(Ultra Search)** 기능을 시연한다. 울트라서치는 GPT타쿠 님의 "인센인 서치(Insane Search)"를 기반으로 X·레딧·쓰레드·해커뉴스·네이버·쿠팡 등 크롤링이 막히는 사이트를 에이전트가 스스로 우회해 검색하고, 결과를 HTML에 실시간 적재한다. 발표자는 오타로 검색이 잘못 흘러가자 **스티어링(steering)**으로 방향을 교정하는 실전 팁까지 보여준다.

## 핵심 개념

- LazyCodex는 코덱스 위에서 도는 하네스여서 **윈도우에서도 문제없이 실행**되고, 설치는 링크의 커맨드 한 줄 복사·붙여넣기로 끝난다.
- `init deep`을 실행하면 에이전트가 프로젝트를 분석해 폴더마다 `AGENTS.md`(에이전트 규칙 파일)를 **중첩(nested)**으로 자동 생성한다.
- `ULW 루프`: 작업이 완수될 때까지 자동 반복. **완결 조건을 프롬프트에 명시**하는 게 중요 — "앱을 개선해 줘"가 아니라 "60프레임 이하로 낮아질 때까지 속도를 최적화해 줘"처럼 측정 가능한 종료 조건을 준다.
- `ULW 플랜`: 토큰은 많이 쓰지만 강력하다. 소크라틱 인터뷰로 요구사항을 캐고, 플래너(프로메테우스/메티스)가 아이디어를 실행 가능한 작업 목록으로 바꾼 뒤 `start work`로 하나씩 실행.
- **울트라서치(Ultra Search)**: 여러 에이전트(시연에서 6개)를 동시에 스폰해 X·레딧·쓰레드 등을 병렬 탐색. 일반 클로드 코드/코덱스로는 막히는 소셜 검색을 우회로를 자동 시도해 뚫는다. 웹서치보다 **훨씬 빠름**.
- HTML에 결과를 **실시간 적재**시키면 사이드바 브라우저로 에이전트가 지금 뭘 하는지 관찰 가능 — "토큰을 써서 사람의 인지 부하를 줄인다"는 철학.
- **스티어링(steering)**: 에이전트가 잘못된 방향(예: 일본 AI 모델을 패션 모델로 오해)으로 갈 때 핸들을 꺾듯 방향을 즉시 교정하는 개입 기능.
- 코덱스는 **핸드폰에서도 사용 가능**해서 출퇴근길·출장 중에 `ULW 루프`로 에이전트에게 작업을 시키는 워크플로가 강점. 초보자에게 특히 추천 — 작업 완수율이 높다.

## 언급된 도구/프레임워크

- `LazyCodex` — 코덱스 위에서 도는 초간단 하네스. OpenCode 제작자(연규)가 만듦. 윈도우 호환.
- `OpenCode` — 연규 님이 만든 기존 하네스. LazyCodex의 지향점.
- `Codex` — OpenAI 코딩 에이전트. LazyCodex의 런타임. 핸드폰 사용 가능.
- `Insane Search / Ultra Search` — GPT타쿠 님이 만든, 소셜/커머스 사이트 크롤링 우회 검색 엔진. LazyCodex에 탑재됨.
- `AGENTS.md` — `init deep`이 폴더별로 자동 생성하는 중첩 에이전트 규칙 파일.
- `프로메테우스 / 메티스` — ULW 플랜의 플래너 에이전트(아이디어 → 실행 가능한 작업 변환).

## 트랜스크립트 발췌

> 오늘 소개시켜 드릴 하네스는 레이지 코덱스라는 하네스입니다. 쓰레드에서 지금 엄청 핫해요. 예전에 OpenCode를 써 보셨으면 아실 텐데, 그 OpenCode를 만드신 연규 님이 이제 코덱스에서도 OpenCode의 느낌이 날 수 있도록 하네스를 제작하셨어요. 기본적으로 사용 방법은 되게 간단합니다. 한 줄이면 설치가 가능하고 심지어 윈도우에서도 정상적으로 동작을 해요. 왜냐면 코덱스에서 돌아가는 하네스기 때문에. 근데 기본적으로 이제 ULW 루프를 이렇게 치게 되면 여러분들이 어떠한 작업을 하고 싶은 거를 쳐 주시면 기본적으로 이 작업이 완수될 때까지 돕니다. 근데 이 완수될 때까지 도는 작업에 여러분들이 조금은 어떤 조건에서 완결이 되는지를 적어 주시면 좋아요. 예를 들면 "앱을 개선해 줘"가 아니라 "앱을 60프레임 이하까지 낮춰질 때까지 속도를 계산해 줘" 이런 식으로 프롬프트를 입력해 주는 게 좋겠죠. (…) 오늘 또 재밌는 게 들어왔어요. 이게 문서에는 없는 거 같지만 오늘 울트라서치라는 게 들어왔어요. GPT타쿠 님의 인센인 서치라는 게 있는데, 인센 서치가 여러분들이 코덱스 쓰실 때 X 그러니까 트위터 아니면 레딧, 쓰레드 이런 데 탐방하기 어렵잖아요. 근데 그거를 어떻게든 우회를 하고 최대한 토큰 최적화를 하셔서 서치를 할 수 있도록 해 줬어요. (…) init deep을 하시면 에이전트가 알아서 AGENTS.md를 폴더 내 만들어 주고요. 근데 여러분들의 폴더 구조가 여러 개가 있을 수 있잖아요. (…) 그런 폴더마다의 규칙을 에이전트가 찾아내서 AGENTS.md라는 파일을 폴더마다 만들어 주게 됩니다. 네스티드에 AGENTS.md라 하는 게 그거를 중첩해서 만들어 준다고 생각하시면 돼요.
