## 5. 적용 우선순위

### P0 (즉시 착수 후보)

**P0-1. VPS + Docker 24/7 전환** (GAP A, E 통합)
- 목표: 집 PC 꺼져도 Bucky `bucky_chat_server(8765)` + `discord_bot` 가동
- 베이스라인: Tailscale ts.net:8443 PWA로 폰 접속만 가능. PC 호스트 종속.
- 타겟: VPS 1대(국내 Cloudtype/Vultr/Cafe24, 월 5-15달러) + Docker compose로 봇 컨테이너 격리
- 마이그레이션 안전: Vault sync는 별도 결정 (G드라이브 호환 vs 별도 git push)
- 검증: PC 종료 상태에서 폰 → Discord 메시지 → Bucky 응답 확인

**P0-2. Memory auto-compaction** (GAP B)
- 목표: BUCKY_CONTEXT.md / Auto-Memory가 일정 크기(예: 50KB) 도달 시 자동 요약 + 주차
- 베이스라인: 무한 누적. 현재 BUCKY_CONTEXT.md ~30KB 도달, 캐시 효율 저하.
- 타겟: `scripts/bucky_memory_compactor.py` — 임계점 초과 시 Gemini로 요약 → 신규 ContextPack `bucky-context-archive-YYYYMM.md`로 분리, 본문은 유지.
- 검증: `wc -c` 측정 + 컴팩션 전후 토큰 수 비교

**P0-3. 작업 후 자동 SKILL.md 추출** (GAP D)
- 목표: 사용자 요청 완료 시 Bucky가 패턴을 자동 추출해 Context Pack 후보 발의
- 베이스라인: Context Pack 수동 작성만 가능
- 타겟: `scripts/bucky_skill_extractor.py` — 완료 핸드오프 5개 누적 시 Gemini-Validator가 공통 패턴 추출 → `06_Context_Packs/_proposals/`에 후보 저장 → 사용자 승인 후 정식 등록
- 검증: 5개 핸드오프 → 1개 후보 파일 생성 확인

### P1 (P0 완료 후)

- **P1-1. SOUL/USER/AGENTS/MEMORY 4파일 분리 + 인터뷰 스크립트** (GAP C) — `scripts/bucky_persona_interview.py`
