# 현장 음성 프라이버시 체크리스트

현장에서 음성 데이터를 수집·STT 변환·Vault 저장 시 개인정보보호법 준수를 위한 필수 점검 항목.

---

## 수집 전 (현장 진입 시)

- [ ] 녹음 전 현장 관계자(고객·협력업체·근로자) 구두 또는 서면 동의 확인
- [ ] 녹음 목적 명확히 고지 ("업무 자동화·AI 보조 도구 활용 목적으로 녹음합니다")
- [ ] 민감 발언 포함 가능성 사전 안내 (이름·전화번호·주소 등)
- [ ] 동의 거부 시 음성 수집 즉시 중단 (텍스트 수동 입력으로 대체)

---

## 수집 중 (녹음 진행 시)

- [ ] 녹음 중 표시 (디스코드 음성 채널 상태 아이콘 또는 구두 고지)
- [ ] 불필요한 개인정보(주민번호·계좌번호·비밀번호) 발화 자제 요청
- [ ] 파일명에 날짜·장소 코드만 사용 (개인 이름 포함 금지)
  - 권장: `siteLog_20260601_gangnam.ogg`
  - 금지: `홍길동_현장_녹음.ogg`

---

## STT 변환 후 (Vault 저장 전)

- [ ] 변환 결과에서 개인정보 마스킹 처리:
  - 이름 → `[이름]`
  - 전화번호 → `[전화번호]`
  - 주소 → `[주소]`
  - 계좌번호 → `[계좌번호]`
- [ ] 원본 오디오 파일은 `external_data/voice_raw/` 에 분리 보관 (Vault 내부 금지)
- [ ] Vault에는 마스킹된 텍스트 트랜스크립트만 저장
  - 저장 경로: `04_SiteLog/{날짜}_{장소코드}_transcript.md`

---

## 보관 및 파기

| 항목 | 기준 |
|------|------|
| 원본 오디오 보관 기간 | 30일 (이후 자동 삭제 또는 수동 파기) |
| 마스킹 트랜스크립트 보관 | 프로젝트 완료 후 1년 |
| 동의서 보관 | 프로젝트 완료 후 3년 |
| 파기 방법 | 파일 완전 삭제 (휴지통 비우기 포함) |

---

## 개인정보처리방침 반영 체크

- [ ] 서비스·앱·웹사이트 운영 시 개인정보처리방침에 "음성 데이터" 항목 추가
- [ ] 수집 목적, 보관 기간, 파기 방법 명시
- [ ] 정보주체 열람·정정·삭제 요청 처리 절차 명시

---

## 자동화 파이프라인 적용 규칙

Discord Bucky 봇이 음성 발화를 수집할 때:

1. `#jh-chat` 음성 채널 입장 시 "음성 기록 활성" 메시지 자동 전송
2. STT 변환 결과는 Vault `04_SiteLog/voice_log_{DATE}.md` 에만 저장
3. 원본 `.ogg` 파일은 `external_data/voice_raw/` 경로에 저장 후 30일 후 삭제 스케줄 등록
4. 마스킹 패턴 자동 적용: `bucky_stt_enhancer.py`의 `_PRIVACY_MASK` 패턴 참고
