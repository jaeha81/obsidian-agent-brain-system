# ADR-0001: 작업 큐 정본 = oracle SQLite 큐 (소급 기록)

- 상태: 승인됨 (사용자, 2026-07-10) — 본 문서는 소급 기록 (작성 2026-07-11, Stage 14)
- 관련: `docs/BUCKY_OS_V3_AUDIT.md` §2·§4 교정 #1

## 맥락

V3 인테이크 문서(07-07)는 `10_AgentBus`에 파일 기반 큐(queue/pending·running·retry·dead_letter) 신설을 가정했다. 그러나 07-10 실측 결과 oracle SQLite + HTTP 큐(`oracle/core/api_server.py` — 상태 7종 전이표, 원자 claim, Bearer 인증)가 이미 구축·테스트 완료(통합 테스트 4종) 상태였다. 두 큐가 병존하면 태스크 상태 정본이 분열된다 (V3 AUDIT 리스크 1위).

## 결정

1. **작업 상태의 정본은 oracle SQLite 큐 하나다.**
2. `10_AgentBus` 신규 파일 큐 신설은 **영구 금지**한다.
3. `10_AgentBus`의 기존 디렉터리(23개, 약 3,379 파일)는 운영 중인 워크플로로 유지하되 정본이 아니다 — 구조 변경은 별도 마이그레이션 계획 필수.

## 결과

- (+) 태스크 상태 단일 조회점, 원자적 선점, 테스트로 보증된 전이.
- (+) Stage 16 레지스트리·Stage 21 대시보드는 큐를 읽기 전용으로 참조하면 된다.
- (−) 큐 retry·DLQ는 미구현 — P1 백로그(P1-5)로 이연.
- 제약: 이후 모든 Stage는 큐를 대체·복제하는 구조물을 만들 수 없다 (분류 축·관측 로그만 허용).
