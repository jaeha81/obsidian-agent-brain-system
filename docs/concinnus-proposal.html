01Executive Summary

강점을 데이터가 쌓이는
고객 여정으로 전환합니다.

콩킨누스는 분위기 있는 브런치·와인 다이닝, 프라임 스테이크와 파스타, 키즈 프렌들리, 프라이빗룸, 대형 화면·음향, 단체·대관 가능성이라는 복합 자산을 보유합니다. 반면 예약·문의·리뷰·운영정보가 여러 채널에 분산되어 있어, 매출 기회를 하나의 고객 데이터로 연결하기 어려운 상태입니다.

i.

매출 성장

단체·기업·기념일·대관 문의를 빠르게 분류하고, 목적·예산·인원에 맞는 맞춤 패키지로 전환합니다.

ii.

운영 효율

반복 문의 응답, 리뷰 정리, 콘텐츠 초안, 일일 매출 보고를 자동화하여 핵심 업무에 집중할 시간을 만듭니다.

iii.

고객 자산화

방문 목적과 선호를 동의 기반으로 축적하여 기념일·기업·가족 세그먼트별 재방문 캠페인을 실행합니다.

06Solution Stack

매출 엔진별 5개 솔루션을

자동화의 성공은 모델 성능보다 "누가 정보를 갱신하고, 누가 승인하며, 어떤 경우 사람이 개입하는지"가 명확할 때 결정됩니다. 각 솔루션은 담당자·승인 절차·검수 주기를 포함합니다.

Solution 01

AI 예약·문의 어시스턴트

단순 질문은 즉시 응답하고, 예약·가격·불만은 정확한 담당자에게 연결합니다. 운영시간과 가격은 AI가 추측하지 않도록 "매장 마스터"에서만 가져옵니다.

대상 문의 분류
  • 즉시 자동응답 · 영업시간, 휴무, 위치, 주차, 유아용 의자, 예약 방법, 룸 기본 정보
  • 정보 수집 후 전달 · 단체 인원·목적·예산·희망 시간·화면·음향·케이터링·법인 결제
  • 직원 승인 필수 · 알레르기 확답, 가격 협상, 불만, 환불, 특수 식단, 대관 계약
처리 흐름 (6단계)
  • 유입 — 네이버 톡톡·카카오·인스타 DM·전화 후 웹폼
  • 분류 — AI가 일반/예약/단체/대관/불만/알레르기로 자동 분류
  • 답변 — 승인된 FAQ는 즉답, 복합 문의는 필요한 추가 질문만 제시
  • 기록·알림·전환 — 리드 생성 → 담당자 알림 SLA 시작 → 결과 기록
Solution 02

리뷰·VOC·재방문 CRM

리뷰를 답글 업무가 아니라 메뉴·서비스 개선과 재방문 신호로 활용합니다. 마케팅 메시지는 수신동의 고객에게만 발송하고, 민감정보는 저장하지 않습니다.

주간 VOC 자동 보고서
  • 수집 — 공식 알림 이메일·플랫폼 내보내기·직원 입력
  • 분석 — 긍정/부정·메뉴·서비스·공간·주차·가격·대기·어린이 태그
  • 요약 — 이번 주 칭찬 3개, 개선 이슈 3개, 재발 이슈, 경쟁 우위
  • 답글 — 브랜드 톤 답글 초안, 낮은 평점·불만은 관리자 승인
재방문 자동화 시점
  • 방문 +1일 — 감사 메시지 및 리뷰 요청
  • 방문 +30일 — 선호 메뉴·목적 기반 신메뉴·와인·브런치 소식
  • 기념일 -14일 — 레터링·룸·와인 사전 예약 안내
  • 기업 분기 1회 — 팀 디너·미팅 패키지, 세금계산서·사전 주문
  • 미전환 단체 +2일 — 빈 시간대 또는 대체 룸·예산 옵션 제안
Solution 03

AI 콘텐츠·로컬 SEO 엔진

사진과 운영정보를 입력하면 채널별 문안과 예약 CTA를 일관되게 생성합니다. AI가 생성한 문안은 반드시 "예약 가능한 실제 상품"과 연결되어야 합니다.

월간 콘텐츠 5대 기둥
  • Signature — 채끝·안심·라자냐·파스타·사워도우의 맛과 재료
  • Space — 룸·스크린·야간 분위기·주차·키즈 프렌들리
  • Occasion — 회식·미팅·기념일·가족모임·대관·촬영
  • People — 셰프·서버·준비 과정·서비스 철학
  • Proof — 리뷰 인사이트, 추천 조합, 고객 질문 답변
로컬 검색 키워드 클러스터
  • 업무·기업 — 판교 회식, 판교 단체식당, 고등동 미팅룸 식당, 판교 와인 회식
  • 가족·브런치 — 고등동 브런치, 판교 아기랑 식당, 주차 편한 브런치
  • 데이트·기념일 — 성남 데이트 맛집, 판교 와인바, 판교 기념일 레스토랑

Solution 04

단체·대관 B2B 세일즈 자동화

공간 자산을 표준 패키지와 빠른 견적으로 매출화합니다. 콩킨누스의 희소 자산인 "대관 가능"을 용도별 패키지·사진·수용 인원·예산·문의 폼으로 표준화합니다.

리드폼 수집 항목
  • 기본 · 희망일·시간·인원·목적·회사·담당자 연락처
  • 예산 · 1인 또는 총예산, 주류, 결제·세금계산서
  • 공간 · 룸·홀, 화면·음향, 좌석배치, 이용·준비시간
  • 식음 · 코스·공유·핑거푸드, 알레르기, 케이터링, 케이크
  • 운영 · 주차 대수, 현수막·촬영, 외부 장비, 계약금·취소
B2B 영업 파이프라인 SLA
  • 신규 리드 → 즉시 접수 메시지 (5분 이내)
  • 유효 확인 → 담당자 알림 (30분 이내)

Solution 05

수요·재고·인력 예측

POS 데이터가 준비되면 메뉴 준비량과 근무 배치를 보조합니다. 예측은 조리 책임자의 판단을 대체하지 않으며, 매일 1분 "운영 보정"이 필요합니다.

예측 산출물
  • 7일 수요 예측 — 요일·시간대별 예상 객수와 매출 범위
  • 메뉴 준비량 — 상위 메뉴별 기본·보수·공격 수량
  • 품절·폐기 경보 — 최근 판매·재고 차이와 이상치 알림
  • 인력 보조 — 예약·예상 매출 기준 홀·주방 권장 인원
도입 4단계
  • 1단계 — 매주 POS CSV 업로드 → 지난주 매출·메뉴 리포트
  • 2단계 — 3 – 6개월 누적 → 다음 주 예측·준비량 제안
  • 3단계 — 재고·폐기 기록 연결 → 폐기율·품절률 최적화
  • 4단계 — 근무표 연결 → 예약·예측 기반 스케줄 보조
