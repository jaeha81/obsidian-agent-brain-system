Threads 자동화 🧵 API 제어

쓰레드 수익화 대시보드를 Discord #jh-쓰레드자동화 채널에서 원격 제어합니다.

§1. 채널 정보

#jh-쓰레드자동화Discord 채널
localhost:3000대시보드 URL (THREADS_DASHBOARD_URL)
Next.js API호출 대상
.env 확인 필요환경변수 상태

§2. 제어 커맨드

Discord 입력API 호출분류
!threads statusGET /api/health상태
!threads runGET /api/cron/threads-daily발행
!threads dry-run?dry_run=true테스트
!threads revenueGET /api/revenue수익
!threads accountsGET /api/accounts계정
[THREADS_CMD] {"action": "status"}JSON 형식 (확장용)JSON

한국어 별칭: !threads 상태, !threads 실행, !threads 테스트, !threads 수익, !threads 계정 도 동일하게 처리된다.

§3. 자연어 위임 동작

특이 동작: 명령어가 아닌 자연어 메시지(!threads/[THREADS_CMD] 접두어 없음)는 이 핸들러가 False를 반환하여 메인 라우터로 위임됩니다. 즉 채널에 그냥 "오늘 발행 잘 됐어?"라고 쓰면 Bucky가 응답합니다.

§4. 필수 환경변수

키설명위치
JH_THREADS_CHANNEL_IDDiscord 채널 ID버키 .env
THREADS_DASHBOARD_URL대시보드 베이스 URL (기본: http://localhost:3000)버키 .env
THREADS_CRON_SECRETcron 보호 Bearer 토큰 (cron 경로 호출 시 자동 첨부)버키 .env
THREADS_DISCORD_WEBHOOK대시보드 → Discord 알림 webhook (선택)프로젝트 .env

§5. 셋업 스크립트

# Discord 채널 + Webhook 자동 생성
python -X utf8 scripts/setup_threads_discord.py

# Webhook 권한 부여
python -X utf8 scripts/grant_webhook_permission.py
