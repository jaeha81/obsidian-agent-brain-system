# 버키에이전트 스킬 도입 가이드

> 출처 영상에서 소개된 5개 스킬을 버키에이전트가 사용할 수 있도록 반영한 기록.
> 버키 = Claude Code CLI 헤드리스 실행 (`scripts/bucky_client.py`, cwd = 이 시스템 루트).

## 반영 결과 요약

| 스킬 | 유형 | 반영 방식 | 상태 |
|---|---|---|---|
| karpathy-guidelines | SKILL.md 1개 | `.claude/skills/`에 직접 배치 | ✅ 완료 |
| watch (claude-video) | SKILL.md + 스크립트 | `.claude/skills/`에 직접 배치 | ✅ 완료 (의존성 별도) |
| superpowers | 플러그인(훅 포함) | Claude Code에서 `/plugin install` | ⬜ 사용자 실행 필요 |
| Understand-Anything | 플러그인(멀티에이전트) | Claude Code에서 `/plugin install` | ⬜ 사용자 실행 필요 |
| agentmemory | 플러그인 + 로컬 서버 | 서버 + `/plugin install` | ⬜ 사용자 실행 필요 |

배치 위치: `G:\내 드라이브\obsidian-agent-brain-system\.claude\skills\`
이 경로는 버키 실행 디렉토리(ROOT)의 표준 프로젝트 스킬 경로라 별도 등록 코드 없이 자동 인식된다.

> ⚠️ **버키 채팅(safe 모드) 제한**: `bucky_chat_once.py`는 `--tools ""`로 실행되어 도구가 없다. 따라서 스킬은 **tools 모드**(`run_bucky_with_tools`, jh-work 작업 채널)에서만 실제로 작동한다. 일반 채팅에는 적용되지 않는다.

## 1. karpathy-guidelines — ✅ 배치 완료

LLM 코딩 실수(과잉 구현·무단 가정·범위 침범)를 줄이는 행동 규칙 4가지. 파일 1개짜리 스킬.

- 위치: `.claude\skills\karpathy-guidelines\SKILL.md`
- 추가 의존성 없음. 코드 작성·리뷰·리팩토링 시 자동 발동.

**버키 채팅에도 항상 적용하고 싶다면** (선택): 루트 `CLAUDE.md`에 규칙을 인라인으로 추가해야 한다. 기존 파일 수정이므로 별도 승인 후 진행.

## 2. watch (claude-video) — ✅ 배치 완료 / ⚠️ 의존성 설치 필요

YouTube·로컬 영상을 다운로드→프레임 추출→자막 추출해 버키가 "영상을 보고" 답하게 하는 스킬.

- 위치: `.claude\skills\watch\` (SKILL.md + scripts/)
- 스크립트는 `${CLAUDE_SKILL_DIR}` 기준으로 자기 폴더를 참조하므로 경로 문제 없음.

**Windows 의존성 (수동 설치 필요 — macOS만 자동)**:

```powershell
# yt-dlp, ffmpeg 설치 (winget 사용 예)
winget install yt-dlp.yt-dlp
winget install Gyan.FFmpeg
```

- 자막 있는 영상은 무료로 동작. 자막 없는 영상만 Whisper API 키 필요.
- Whisper 키가 필요해지면 `~/.config/watch/.env`에 저장 (스킬이 안내).
- ⚠️ Windows에서는 SKILL.md의 `python3` 명령을 `python`으로 치환해야 함 (스킬 문서에 명시되어 있음).

사용 예 (버키 tools 모드 / Claude Code 직접):

```
/watch https://youtu.be/dQw4w9WgXcQ 30초 지점에 뭐가 나오나?
```

## 3. superpowers — ⬜ Claude Code에서 설치

스펙→계획→TDD를 강제해 첫 결과물 품질을 끌어올리는 워크플로우 플러그인. 훅이 포함돼 파일 복사만으로는 작동하지 않으므로 Claude Code 안에서 설치한다.

```bash
# Anthropic 공식 마켓플레이스 (권장)
/plugin install superpowers@claude-plugins-official
```

또는 저자 마켓플레이스:

```bash
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
```

> 트레이드오프: 간단한 작업도 스펙·계획·테스트 단계를 거쳐 느려질 수 있음. 사용자의 기존 "소/중/대 규모" 원칙과 충돌 시 소규모 작업엔 비활성 판단.

## 4. Understand-Anything — ⬜ Claude Code에서 설치

코드베이스 전체를 멀티에이전트로 스캔해 지식 그래프·인터랙티브 대시보드를 만드는 플러그인.

```bash
/plugin marketplace add Lum1104/Understand-Anything
/plugin install understand-anything
```

설치 후 프로젝트 폴더에서:

```bash
/understand                    # 분석 시작 → .understand-anything/knowledge-graph.json 생성
/understand dashboard          # 인터랙티브 대시보드 열기
/understand --language ko      # 한국어 출력
```

코드뿐 아니라 PDF·마크다운·이미지도 비전으로 읽어 그래프에 포함. 옵시디언 지식베이스 정리에도 활용 가능.

## 5. agentmemory — ⬜ 서버 + 플러그인 설치

세션 작업을 자동 기록하고 벡터·그래프 검색으로 다음 세션에 필요한 컨텍스트만 주입하는 메모리 시스템. **로컬 서버 상시 실행이 전제.**

```bash
# 1) 별도 터미널에서 메모리 서버 실행 (Node 필요)
npx -y @agentmemory/agentmemory@latest

# 2) Claude Code에서 플러그인 설치
/plugin marketplace add rohitg00/agentmemory
/plugin install agentmemory

# 3) 동작 확인
curl http://localhost:3111/agentmemory/health
# 실시간 뷰어: http://localhost:3113
```

> ⚠️ **버키 환경 주의점 2가지**
> 1. 버키는 `--no-session-persistence`로 실행되어 세션 훅 기반 자동 기록 효과가 제한될 수 있음. 도입 효용 대비 운영 부담(서버 상시 실행·Node 환경)이 가장 큼 → **마지막 우선순위 권장.**
> 2. 플러그인 없이 MCP만 직접 연결하면 버전 경로가 박혀 업그레이드 시 훅이 깨짐(이슈 #508). 반드시 `/plugin install` 경로 사용.
